# End To End: Finite Modules

This lesson follows one finite module resource from ring-action tables to
replayed result and proof/evidence status. It uses the
[finite-modules-v0](../../../artifacts/examples/math/finite-modules-v0/) pack.

Concept rows:

- `curriculum_rings`, `curriculum_groups`, and `curriculum_linear_algebra` in

- `field_abstract_algebra`, `field_linear_algebra`, and

The replay rows are exact finite table checks over `Z/4Z`. The pack does not

projective or injective modules, Noetherian module structure, or homological
algebra.

## Encode

The scalar ring is `Z/4Z`:

```text
R = {0, 1, 2, 3}
```

The module carrier is also `Z/4Z`, with module addition computed modulo `4`.
Scalar multiplication is ring multiplication modulo `4`:

```text
2 * 1 = 2
2 * 2 = 0
3 * 3 = 1
```

This is the regular module: the ring acts on its own additive group.

## Replay The Module Laws

The checker first validates the finite ring table and the additive abelian
group table. Then it checks the left-module laws over every listed scalar and
module element:

```text
1*m = m
0*m = 0
r*0 = 0
r*(m + n) = r*m + r*n
(r + s)*m = r*m + s*m
(r*s)*m = r*(s*m)
```

For example:

```text
2*(1 + 3) = 2*0 = 0
2*1 + 2*3 = 2 + 2 = 0
```

The validator repeats this finite replay across the whole table.

## Replay A Submodule And Span

The submodule candidate is the two-torsion subset:

```text
N = {0, 2}
```

The span witness says `N` is generated by `2`:

```text
0*2 = 0
1*2 = 2
2*2 = 0
3*2 = 2
span(2) = {0, 2}
```

The checker verifies zero membership, additive closure, scalar closure, and
equality with the recomputed generated submodule.

## Replay A Module Homomorphism

The finite homomorphism is multiplication by `2`:

```text
f(0) = 0
f(1) = 2
f(2) = 0
f(3) = 2
```

The checker verifies additive preservation and scalar preservation:

```text
f(m + n) = f(m) + f(n)
f(r*m) = r*f(m)
```

One scalar row is:

```text
f(3*3) = f(1) = 2
3*f(3) = 3*2 = 2
```

Then it recomputes:

```text
kernel(f) = {0, 2}
image(f)  = {0, 2}
```

The kernel and image are not trusted because the producer listed them; they are
derived from the finite map.

## Replay The Quotient Module

The quotient by `{0, 2}` has two cosets:

```text
E = {0, 2}
O = {1, 3}
```

The quotient addition table is:

```text
E + E = E
E + O = O
O + O = E
```

The scalar action table records:

```text
0*E = E, 0*O = E
1*E = E, 1*O = O
2*E = E, 2*O = E
3*E = E, 3*O = O
```

The checker verifies these by applying module addition or scalar multiplication
to representatives, then locating the resulting coset. For example:

```text
3 * O uses representative 1
3 * 1 = 3, and 3 in O
```

So `3*O = O`.

## Check The Refutation

The bad row claims that this subset is a submodule:

```text
B = {0, 1}
```

Scalar closure fails:

```text
2 * 1 = 2
```

Because `2` is not in `{0, 1}`, the subset is not closed under scalar
multiplication and cannot be a submodule. The linked `QF_UF` artifact records
`1` present in the claimed subset, `2` absent, `2 * 1 = 2`, and the fixed
scalar-closure membership claim `in_subset(smul(2,1)) = present`; Axeyum emits
and independently rechecks an `UnsatAletheProof` for that equality conflict.

## Trust Boundary

This lesson shows Axeyum's resource pattern for module theory over a finite
ring:

```text
untrusted fast search -> module table, submodule, homomorphism, quotient table
trusted small checking -> action laws, span replay, kernel/image, quotient replay
checked proof object -> QF_UF/Alethe certificate for the bad submodule row
```

General module theory, quotient-module theorems, exact sequences, tensor
products, projective and injective modules, Noetherian modules, and homological
algebra require Lean/mathlib-scale proof support.
