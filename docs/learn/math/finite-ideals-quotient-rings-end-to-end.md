# End To End: Finite Ideals And Quotient Rings

This lesson follows one finite ideal resource from modular-ring table data to
replayed result and proof/evidence status. It uses the
[finite-ideals-v0](../../../artifacts/examples/math/finite-ideals-v0/) pack.

## Encode

The ambient ring is `Z/6Z`:

```text
R = {0, 1, 2, 3, 4, 5}
```

Addition and multiplication are the usual operations modulo `6`. The candidate
ideal is the even residue set:

```text
I = {0, 2, 4}
```

The same set is also represented as the principal ideal generated by `2`:

```text
(2) = {r * 2 | r in Z/6Z} = {0, 2, 4}
```

## Replay Ideal Closure

The checker first validates the finite ring tables. Then it checks the ideal
obligations over all listed ring and ideal elements:

```text
0 in I
a, b in I -> a + b in I
a in I -> -a in I
r in R, a in I -> r*a in I
r in R, a in I -> a*r in I
```

For example:

```text
2 + 4 = 0 mod 6
5 * 2 = 4 mod 6
2 * 5 = 4 mod 6
```

All three results are still in `I`. The validator repeats this finite
calculation for every table row, so the ideal claim is trusted by replaying the
small table rather than by trusting the producer of the witness.

## Replay Principal Generation

The principal-ideal row recomputes the closure generated by `2`. Multiplying
the generator by every ring element gives:

```text
0*2 = 0
1*2 = 2
2*2 = 4
3*2 = 0
4*2 = 2
5*2 = 4
```

The generated set is exactly `{0, 2, 4}`. The checker compares that recomputed
set with the listed ideal.

## Replay Kernel And Image

The pack also encodes reduction modulo `2` as a unital ring homomorphism:

```text
0 -> 0
1 -> 1
2 -> 0
3 -> 1
4 -> 0
5 -> 1
```

The validator checks preservation of zero, one, addition, and multiplication.
Then it recomputes:

```text
kernel = {x in Z/6Z | f(x) = 0} = {0, 2, 4}
image  = {f(x) | x in Z/6Z} = {0, 1}
```

This ties the ideal back to a familiar algebraic pattern: the even residues are
the kernel of the parity map.

## Replay The Quotient Ring

The quotient by `I` has two cosets:

```text
E = {0, 2, 4}
O = {1, 3, 5}
```

The listed quotient operations are the `Z/2Z` tables:

```text
E + E = E
E + O = O
O + O = E

E * E = E
E * O = E
O * O = O
```

The checker verifies that the cosets partition `Z/6Z`. It then applies ring
addition and multiplication to representatives and locates the resulting coset.
For example:

```text
1 + 3 = 4 mod 6, and 4 in E
3 * 5 = 3 mod 6, and 3 in O
```

So `O + O = E` and `O * O = O`. The finite quotient table is accepted only if
all representative calculations agree with the listed result.

## Check The Refutation

The bad row claims that `{0, 2}` is an ideal of `Z/6Z`. The checker rejects it
by recomputing one additive-closure failure:

```text
2 + 2 = 4 mod 6
```

Because `4` is not in `{0, 2}`, the subset cannot be an additive subgroup and
therefore cannot be an ideal. The linked `QF_UF` artifact records `2` present
in the claimed subset, `4` absent, `2 + 2 = 4`, and the fixed additive-closure
membership claim `in_subset(add(2,2)) = present`; Axeyum emits and
independently rechecks an `UnsatAletheProof` for that equality conflict.

## Trust Boundary

This lesson shows Axeyum's resource pattern for quotient algebra:

```text
untrusted fast search -> candidate ideal, homomorphism, cosets, quotient tables
trusted small checking -> ring table replay, ideal closure, kernel/image,
                          quotient replay
checked proof object -> QF_UF/Alethe certificate for the bad ideal row
```

General ideal theory, correspondence theorems, the first isomorphism theorem
for arbitrary rings, prime and maximal ideals, localization, Noetherian ideal
theory, spectra, sheaves, and algebraic geometry require Lean/mathlib-scale
proof support.
