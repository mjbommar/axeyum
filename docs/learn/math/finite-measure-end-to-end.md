# End To End: Finite Measure

This lesson follows one finite measure resource from sigma-algebra replay to
exact finite additivity, event complements, and checked rejection of a malformed
complement-measure row. It uses the
[finite-measure-v0](../../../artifacts/examples/math/finite-measure-v0/)
pack.

Concept rows:

- `field_measure_theory`, `field_probability_theory`, and
  `field_set_theory_and_foundations` in the
  [Foundational Concept Atlas](../../../artifacts/ontology/foundational-concepts.json)
- `curriculum_sets`, `curriculum_rationals`, and `curriculum_counting` in the
  [math coverage dashboard](../../foundational-resources/generated/math-coverage.md)
- `bridge_probability_mass_table`, `bridge_finite_boolean_algebra`, and
  `family_exact_rational_farkas` in the atlas bridge/example-family vocabulary.

## Claim Shape

| Check | Expected | Evidence Status |
|---|---|---|
| `finite-sigma-algebra-axioms` | `sat` | replay-only |
| `finite-measure-additivity` | `sat` | replay-only |
| `event-complement-measure` | `sat` | replay-only |
| `bad-complement-measure-rejected` | `unsat` | checked QF_LRA/Farkas |

All rows are finite and exact-rational. The pack checks a finite
sigma-algebra, a normalized finite measure table, and one bad complement claim.
It does not prove countable additivity, Lebesgue measure, convergence theorems,
or general measure-theory facts.

## Encode

The finite sigma-algebra is generated by a two-block partition:

```text
universe = {a,b,c,d}
measurable sets = {}, {a,b}, {c,d}, {a,b,c,d}
```

The exact measure table is:

```text
mu({}) = 0
mu({a,b}) = 1/3
mu({c,d}) = 2/3
mu({a,b,c,d}) = 1
```

For the event-complement row:

```text
E = {a,b}
E^c = {c,d}
mu(E) = 1/3
mu(E^c) = 2/3
mu(U) = 1
```

## Replay

The sigma-algebra replay checks:

```text
{} and U are measurable
complements of measurable sets are measurable
pairwise unions of measurable sets are measurable
```

The finite measure replay checks nonnegativity, normalization, and finite
additivity on disjoint measurable sets. For the two partition blocks:

```text
{a,b} disjoint {c,d}
mu({a,b}) + mu({c,d}) = 1/3 + 2/3 = 1
mu({a,b,c,d}) = 1
```

For the complement row, the checker recomputes:

```text
complement({a,b}) = {c,d}
mu({a,b}) + mu({c,d}) = 1/3 + 2/3 = 1
```

## Check The Refutation

The promoted bad row keeps the finite source object fixed but changes the
claimed complement measure:

```text
mu(E) = 1/3
claimed mu(E^c) = 1/2
mu(U) = 1
mu(E) + mu(E^c) = mu(U)
```

Finite replay computes `mu(E) = 1/3` and `mu(U) = 1`. The committed SMT-LIB
artifact
[`bad-complement-measure-farkas-conflict.smt2`](../../../artifacts/examples/math/finite-measure-v0/smt2/bad-complement-measure-farkas-conflict.smt2)
checks the final exact-rational contradiction:

```text
1/3 + 1/2 = 1
```

The solver search is untrusted. The accepted evidence is rechecked
`UnsatFarkas` arithmetic over the source artifact.

