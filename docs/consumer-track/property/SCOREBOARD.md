# App B — property SDK SCOREBOARD

> Self-contained, construction-known bounded-property scoreboard for the
> `axeyum-property` SDK (App B), generated by `axeyum-consumer-bench` (App D).
> Ground truth is **by construction** (each property is authored with a known
> true status), so **no external oracle is needed**. The hard soundness floor
> is **DISAGREE = 0**: axeyum's verdict must never *contradict* the
> construction-known status (`Unknown` is allowed, never a contradiction).
>
> Regenerate with `scripts/gen-property-scoreboard.py` (re-runs the binary).

### Notes

- *construction-known*: the authored true status — `should-prove` (a valid
identity) or `should-find-ce` (a deliberate bug with a named witness).
- *match*: `yes` = axeyum's decided verdict equals the known status;
`unknown` = axeyum honestly did not decide (allowed); `DISAGREE` = a
contradiction (a hard failure — must never appear).
- *cert re-checked*: for a `proved` row, `Certificate::verify()` independently
re-validated the evidence.
- *Lean module*: for a `proved` row, `Certificate::to_lean_module()` emitted a
standalone Lean module (the differentiator; `no` is honest, never faked).
