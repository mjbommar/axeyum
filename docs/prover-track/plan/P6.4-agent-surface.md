# P6.4 — Agent surface

**Size:** M · **Depends on:** P6.2 (not P6.3) · **Falsifies the surface claim**

## Goal
Expose the goal layer to an agent as **data and forkable states**, not text — and
find out whether that is worth anything.

## Why / leverage
Reachable straight after P6.2, over *existing* automation. That ordering is
deliberate: this phase is the cheapest place to learn the surface is not the
product.

**The structural advantage nobody noticed for four drafts**: *Keep the Proof State
Live* measures **~99.9% of agent per-branch wall time as import + re-elaboration**
(~60 s import; tactic execution **<0.1%**). **We have no library to import.** The
field's entire cost model is a tax we do not pay — bigger than the WASM claim, and
it falls out of refusing to build a Mathlib.

## Phase exit / falsification — the experiment must not be rigged
Run the surface against a **`lean4check`-shaped loop** (a checker and a retry loop,
one tool) on:

- a **named** obligation set, fixed **in advance**;
- **≥100** obligations;
- **search-heavy** — requiring exploration, not repair.

**Both branches carry the same burden of proof.** v2's version had a "no" branch
with no qualifiers and permanent effect, and a "yes" branch needing "decisively"
and "search-heavy". Symmetric, or it is theatre.

**K1 fires** if the loop lands within **5 points** → the rich surface is not the
product. **Revisit only on new evidence** — *not* "permanently"; that word was
doing rhetorical work.

### Calibration, to prevent overselling
`lean4check` + Claude Code reaches **87% on 189 proof-engineering tasks with one
tool**. AxProverBase's own ablation ranks **iterative refinement ≫ memory ≫ tools
("marginal")**. A rich surface pays on **search-heavy** work only. **Do not sell the
mechanical case.**

## Ship the tool regardless — but that is not entering the rung
Note 04: *"Not one of them builds their own solver. All of them rent Z3."* The MCP
server is picks-and-shovels and cheap, so **T6.4.1-3 ship whatever K1 says**.

**This is not in tension with [ADR-0167](../../research/09-decisions/adr-0167-prover-track-entry.md)'s
"any gate failing closes the rung"** — round 4 caught that contradiction and it is
resolved by the distinction: **shipping a tool is not entering the rung.** If K1
fires, the MCP server ships and *everything above P6.1a stops.*

## The verification boundary (added by this phase)
- **Newly in the TCB:** nothing. The agent is untrusted by construction; that is
  the entire design.
- **Newly exposed:** every decline reason becomes public API. Say what is *not*
  covered in the same breath — an agent that cannot tell "refuted" from "you
  encoded nonsense" will report the second as the first.
