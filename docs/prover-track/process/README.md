# Process record — not the deliverable

**Nothing in this folder is a plan.** It is the record of how the plan was arrived
at: seven drafts, four adversarial reviews, seven premises that were wrong.

**If you want to build the thing, you want:**

- [`../design/03-architecture.md`](../design/03-architecture.md) — what it is
- [`../plan/README.md`](../plan/README.md) — how to build it
- [`../REFERENCES.md`](../REFERENCES.md) — the sources

**Read this folder only if** you are about to re-litigate a decision, or you want
to know why a constraint in the plan is there. Every constraint traces to something
here.

| File | What |
|---|---|
| [`DIARY.md`](DIARY.md) | Append-only log. Scored priors, dead ends, seven corrections. |
| [`critique/`](critique/) | Four independent adversarial reviews. **Their findings became the plan's constraints; their verdicts on *whether* to build are superseded.** |
| [`options-considered.md`](options-considered.md) | The design options and why each was taken or refused. |
| [`quantified-uf-probe.md`](quantified-uf-probe.md) | Running the five quantified-UF goals, and the two corrections it produced. |

## The one thing here worth reading anyway

> **Provenance attaches to claims, not to documents.**

Seven premises died in this track. Every one was a number or a term, correctly
quoted, whose *meaning* was assumed rather than traced to the code or the model
theory that produced it. The most expensive was a claim in a *code audit* that came
from reading a **scoreboard slice name** — trusted by association with the
document's provenance rather than its own.
