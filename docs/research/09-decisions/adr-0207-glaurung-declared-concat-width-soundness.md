# ADR-0207: Glaurung declared-concat-width soundness

Status: accepted
Date: 2026-07-16

## Context

ADR-0206 required exact formulas before interpreting the widened `tcpip`
nondecisions. Glaurung's strict 60-second capture validates 784 distinct tcpip
split formulas / 234,463,502 bytes. Of the 736 formulas Z3 decides and Axeyum
does not, 733 are Axeyum errors and only three are Axeyum `Unknown`; Axeyum
decides the other 48 while Z3 is nondecided. The corresponding `dxgkrnl` pack
has four distinct formulas, including two Axeyum errors.

Axeyum's byte-owning manifest replay reduces both the SAT and UNSAT error
families to the same precise rejection: `extract [63:8] out of range for width
57`. Glaurung's expression declared a concat as 56 high bits plus eight low
bits, but its low child was a one-bit `setcc` value. The SMT renderer and both
native adapters ignored the declared concat child widths, constructing 57 bits
while the expression pool and concrete domain correctly recorded 64. Z3's
later extract coercion hid the mismatch and shifted the high half by one bit
rather than eight, so this was a downstream soundness defect, not an Axeyum SAT
failure.

## Decision

Accept Glaurung ADR-016/`d60ed0f`. Before concat, its SMT-LIB renderer, Z3
adapter, and Axeyum adapter now coerce each child to the node's declared
`hi_w`/`lo_w` by zero-extension or low-bit truncation. This implements the
existing `Domain::concat(hi, lo, hi_w, lo_w)` contract and matches concrete
execution.

Keep Axeyum's strict sort checks and precise `IrError`; add no permissive concat
surface. Preserve both old malformed and post-fix residual split packs in
Glaurung Git LFS with strict TSV validation, hash-free capture indexes, and
Axeyum-generated byte manifests.

## Evidence

Focused Glaurung renderer, Z3, and native Axeyum tests agree that a one-bit low
child declared as eight bits produces the 64-bit value `0x1201`; all 43 Axeyum
backend tests pass under the 4 GiB cap.

The identical post-fix 60-second policies remove every adapter error and warm
reset:

- `tcpip`: 72,291 queries, zero SAT/UNSAT disagreements, resets 925→0,
  Axeyum nondecisions 931→15, split occurrences 977→55, 1.9x speedup, and
  447,888 KiB RSS. Its 52 distinct residual formulas contain 43 Axeyum-only
  decisions and nine Z3-only decisions.
- `dxgkrnl`: 17,712 queries, zero SAT/UNSAT disagreements, resets 2→0,
  Axeyum nondecisions 2→0, three split occurrences, 2.7x speedup, and
  341,732 KiB RSS. Both distinct residual formulas are Axeyum SAT versus Z3
  `Unknown`.

Axeyum's independent cold runner decides all nine Z3-decided residual tcpip
formulas under a 30-second diagnostic cap (1 SAT / 8 UNSAT, 9/9 manifest
agreement). SAT search is 93.2% of pipeline time and total p50/max are
213.7/399.0 ms. Under Glaurung's 250 ms cap, 5/9 decide and four return explicit
`Unknown(Timeout)`. The residual is therefore a bounded SAT-timeout policy
class, not unsupported syntax, translation failure, or a wrong verdict.

## Alternatives

- Coerce only in the Axeyum adapter: rejected because text capture, Z3, and
  concrete execution would still disagree.
- Make Axeyum concat permissive: rejected because it would hide the consumer
  defect and weaken a useful framework invariant.
- Fix only `setcc`: rejected because any declared-width concat child could
  reproduce the same inconsistency.
- Count residual timeout splits as agreement: rejected because `Unknown`
  remains first-class and changes exploration completeness.

## Consequences

The old packs remain exact soundness reproducers, not performance evidence.
The post-fix packs are the valid timeout-diagnostic boundary. `tcpip` and
`dxgkrnl` still require repeated findings/RSS/variance gates (and the standard
full-budget tier) before DriverSpec admission. The immediate client task is to
measure warm-state versus cold 250 ms behavior and choose an explicit,
replay-safe timeout/fallback policy; broad lowering work is not selected by
these nine SAT-dominated formulas.
