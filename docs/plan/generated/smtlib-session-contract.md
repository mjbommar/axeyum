# SMT-LIB session contract prototype

> **Generated; do not edit by hand.** Source: [`docs/plan/smtlib-session-contract-v1.json`](../smtlib-session-contract-v1.json). Regenerate with `python3 scripts/gen-smtlib-session-contract.py`; use `--check` in validation.

This is an executable planning model for command ordering, modes, options, declaration scope, query snapshots, output routing, reset, and error atomicity. It does not solve formulas and is not production SMT-LIB code.

## Current implementation findings

| Finding | Current source fact | Consequence |
|---|---|---|
| `F1-narrow-command-ir` | `crates/axeyum-smtlib/src/parse.rs`: ScriptCommand records assertion-stack operations and get-assertions, not the complete protocol stream. | Output/lifecycle commands cannot be associated with their exact state or preceding query. |
| `F2-final-global-option-maps` | `crates/axeyum-smtlib/src/parse.rs`: Option updates collapse into one final map while get-option requests live in a separate list. | A request can observe a later value instead of the value at its command point. |
| `F3-output-noops` | `crates/axeyum-smtlib/src/parse.rs`: Several standard output commands are syntax-only no-ops. | Parser acceptance currently overstates executable command conformance. |
| `F4-full-reset-rejected` | `crates/axeyum-smtlib/src/parse.rs`: Full reset is rejected because every parsed term shares one arena. | Production reset requires a fresh arena/session epoch, not clearing vectors in place. |
| `F5-declarations-always-global` | `crates/axeyum-smtlib/src/parse.rs`: Declarations stay global across push/pop regardless of the standard global-declarations default. | The current incremental helper is assertion-stack equivalent, not protocol/signature equivalent. |
| `F6-pop-underflow-silent` | `crates/axeyum-solver/src/smtlib.rs`: Pop consumes only available scopes and silently ignores underflow. | A malformed command changes neither output nor error state as SMT-LIB requires; the runner needs transactional validation. |
| `F7-session-types-absent` | `crates/axeyum-solver/src/smtlib.rs`: No public session state or typed response contract exists. | The new boundary can be designed without preserving an accidental public session ABI. |
| `F8-unknown-has-no-model` | `crates/axeyum-solver/src/backend.rs`: CheckResult carries a model only for Sat, while SMT-LIB places both sat and unknown in its sat query mode. | The session needs an explicit deterministic model-inspection policy for unknown; mode legality alone cannot manufacture a replayed model. |

## Prototype boundary

The reference model intentionally does not parse terms, invoke a solver, serialize models/proofs, or write files. Production work must preserve these control-plane traces while replacing placeholder payloads such as `query-1` with replay-checked typed results and canonical SMT-LIB rendering.
