E1/E2/E3 cell evidence to external result publication. This increment used
only temporary synthetic fixtures. It did not probe a host, run a sentinel or
solver, create a resource session or allocation, stop a systemd unit, or read
or mutate a live NAS run root.

`full_coordinator.py` replays, in order:

1. the immutable F2 preparation while permitting runtime additions only under
   an exact solver-order prefix;

   wave checkpoints;

   terminal;
4. the complete E1 bundle, output sidecars, resource completion, and E3
   multi-host completion;
5. every selection row and v2 result record, including exact sequence,
   benchmark, expected-status, and per-logic accounting; and

Only after those checks does it derive the execution authority consumed by the
completion-last cell-result publisher. The composed publication entry point
always performs that replay before installing result bytes.

## Correctness repairs found during the audit

The coordinator audit exposed four prerequisites that the earlier fixture did
not cover:

- strict F2 replay rejected its own run after controlled execution began;
  replay now admits additions only under `axeyum`, then `axeyum` plus `cvc5`,
  then all three cells, never a reordered or skipped prefix;
- checkpoint schema v1 could represent only three successful initial
  allocations, so a preregistered one-shard retry could never rejoin its wave;
  schema v2 records six shard completions and accepts only each shard's exact
  initial owner or exact registered different-host retry;
- checkpoints previously existed only as returned Python objects; they now
  install immutably as a contiguous prefix, quarantine interrupted temporaries,
  and reject gaps, replacements, and extra artifacts; and
- cell adjudication stopped on known-status contradictions but not prior-solver
  disagreement; execution authority v2 now binds the exact prior-result prefix
  and recomputed disagreement count, and adjudication withholds continuation
  on either condition.
