# P3.8 — Craig interpolation (proof-based, theory-aware)

**Size:** L · **Depends on:** [P3.2 Alethe IR](P3.2-alethe-ir.md),
[P3.5 reduction proofs](P3.5-reduction-proofs.md); LRA Farkas (in tree) ·
**New capability · enables P4.6**

## Goal
Given an `unsat` conjunction `A ∧ B`, produce a **Craig interpolant** `I`:
`A ⇒ I`, `I ∧ B ⇒ ⊥`, and `I` uses only symbols shared by `A` and `B`. Expose it
as the SMT-LIB `(get-interpolant)` command and as a Rust API. Because axeyum
already produces *checked* refutations, the interpolant is read off the **proof**,
not re-derived by a separate untrusted procedure — and it is itself re-checkable
(the three implications are independent `unsat` checks).

## Why / leverage
Interpolation is a first-class feature in Z3 and cvc5 and a missing column here.
More importantly it is the **lemma engine for CHC/PDR** (P4.6): generalizing a
blocked counterexample frame is exactly an interpolation query. Building
interpolation as a proof transform (not a bespoke solver) keeps it inside the
trust story — the interpolant is a *consequence of an already-checked proof*, so
it inherits the assurance instead of adding a new trusted component.

## Design notes
- **Propositional / BV core:** interpolate the resolution (DRAT/LRAT) refutation
  by the McMillan/Pudlák system — partition leaves by `A`/`B`, propagate partial
  interpolants through resolution. axeyum already has the checked clausal proof
  (P3.1/`check_drat`), so this is a transform over an object we trust.
- **LRA:** interpolate the **Farkas** certificate directly (the `A`-part linear
  combination is the interpolant). axeyum already emits and verifies Farkas
  certificates — this is the cheapest, highest-assurance first slice.
- **EUF:** interpolate the congruence-closure explanation (P2.3/`check_congruence`)
  via the standard coloring of the explanation DAG.
- **Combination:** Nelson–Oppen-style interpolation over the equality-sharing bus
  (P1.6) once EUF+LRA interpolation each work.
- **Contract:** every produced interpolant is verified by three independent
  `unsat`/entailment checks before it is returned; on any doubt, decline
  (`unknown`/unsupported), never return an unverified interpolant.

## Implementation notes (2026-06-22)

Landed and in progress (ADR-0047, verify-before-return throughout):

- **T3.8.1 LRA — DONE.** `axeyum_solver::lra_interpolant` reads the interpolant off
  the self-checked Farkas certificate (`FarkasCertificate` gained a `vars` field);
  A-only variables cancel automatically, so the shared-vocabulary condition is free.
- **T3.8.3 EUF — DONE.** `axeyum_solver::qf_uf_interpolant` summarizes the
  congruence-closure `explain_steps` path, coloring edges by partition and lowering
  non-shared congruence boundaries to argument equalities.
- **T3.8.5 façade — partial.** `Solver::interpolant(arena, a_indices)` dispatches
  LRA → EUF. SMT-LIB `(get-interpolant)` *parse* surface deferred (coordinate
  `axeyum-smtlib`); the solver-side driver can land without it.
- **T3.8.2 propositional/BV — substrate mapped.** Build the propositional
  interpolant over the **LRAT antecedent-hint chains** (`axeyum_cnf::lrat`), not the
  Alethe `resolution` rule (which is entailment-checked and discards pivots). Each
  `LratStep::Add{clause, hints}` is a RUP chain; replaying the forward propagation
  recovers the pivot per hint, giving a genuine resolution derivation to fold
  McMillan's partial interpolants over. Input clause id `i ≤ |A|` ⇒ A-side, else B.
  The CNF carries **no per-clause provenance**, so partition labels come from
  unioning the two formulas (A's clauses first) — at the propositional level the
  caller supplies `A`/`B` as separate `CnfFormula`s over a shared variable space.
  Slice 1 (in progress): a self-contained `axeyum_cnf::propositional_interpolant`
  returning a `BoolExpr` over the global (shared) variables, verified by SAT.
  **BV-term lifting (T3.8.2b, in progress):** the alignment strategy is **single
  joint lowering** — `lower_terms(arena, A ++ B)` then one `tseitin_encode`. AIG
  structural hashing + per-`TermId` memoization make a shared term/symbol bit
  collapse to one `AigLit` ⇒ one `CnfVar` across both sides (separate lowerings do
  NOT canonicalize input ids by symbol, so they desync). Partition the single CNF's
  *clauses* by per-root reachability (`reachable_node_mask`, made `pub`): a clause
  goes to side S iff one of its vars' AIG nodes is reachable from S's roots; a
  **clause reachable from both sides goes into both** (duplication is sound and is
  what makes a genuinely-shared bit a *global* var). Run
  `propositional_interpolant(a_cnf, b_cnf)` over the shared variable space, then lift
  each global `CnfVar` → `AigLit` (`variable_bindings`) → `(TermId, bit)` (scan
  `symbol_inputs`/`term_bits`) → the predicate `((_ extract i i) t) = #b1` (negated
  if the term-bit literal is inverted). Verify-guarded by the QF_BV decider on
  `A ∧ ¬I` and `I ∧ B` + shared-symbol vocabulary; decline if any interpolant var
  doesn't map to a shared-term bit.

## Phase exit criteria
- `(get-interpolant)` works for QF_LRA, QF_BV, QF_UF, and conjunctive UFLRA, with
  **every** returned interpolant verified by independent entailment/`unsat`
  checks before return.
- The CHC engine (P4.6) consumes the interpolation API for frame generalization.
- No unverified interpolant ever escapes; unsupported fragments return a clean
  unsupported/`unknown`, never a guess.
