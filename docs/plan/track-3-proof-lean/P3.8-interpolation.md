# P3.8 ‚Äî Craig interpolation (proof-based, theory-aware)

**Size:** L ¬∑ **Depends on:** [P3.2 Alethe IR](P3.2-alethe-ir.md),
[P3.5 reduction proofs](P3.5-reduction-proofs.md); LRA Farkas (in tree) ¬∑
**New capability ¬∑ enables P4.6**

## Goal
Given an `unsat` conjunction `A ‚àß B`, produce a **Craig interpolant** `I`:
`A ‚áí I`, `I ‚àß B ‚áí ‚ä•`, and `I` uses only symbols shared by `A` and `B`. Expose it
as the SMT-LIB `(get-interpolant)` command and as a Rust API. Because axeyum
already produces *checked* refutations, the interpolant is read off the **proof**,
not re-derived by a separate untrusted procedure ‚Äî and it is itself re-checkable
(the three implications are independent `unsat` checks).

## Why / leverage
Interpolation is a first-class feature in Z3 and cvc5 and a missing column here.
More importantly it is the **lemma engine for CHC/PDR** (P4.6): generalizing a
blocked counterexample frame is exactly an interpolation query. Building
interpolation as a proof transform (not a bespoke solver) keeps it inside the
trust story ‚Äî the interpolant is a *consequence of an already-checked proof*, so
it inherits the assurance instead of adding a new trusted component.

## Design notes
- **Propositional / BV core:** interpolate the resolution (DRAT/LRAT) refutation
  by the McMillan/Pudl√°k system ‚Äî partition leaves by `A`/`B`, propagate partial
  interpolants through resolution. axeyum already has the checked clausal proof
  (P3.1/`check_drat`), so this is a transform over an object we trust.
- **LRA:** interpolate the **Farkas** certificate directly (the `A`-part linear
  combination is the interpolant). axeyum already emits and verifies Farkas
  certificates ‚Äî this is the cheapest, highest-assurance first slice.
- **EUF:** interpolate the congruence-closure explanation (P2.3/`check_congruence`)
  via the standard coloring of the explanation DAG.
- **Combination:** Nelson‚ÄìOppen-style interpolation over the equality-sharing bus
  (P1.6) once EUF+LRA interpolation each work.
- **Contract:** every produced interpolant is verified by three independent
  `unsat`/entailment checks before it is returned; on any doubt, decline
  (`unknown`/unsupported), never return an unverified interpolant.

## Implementation notes (2026-06-22)

**The interpolation engine is DONE for every phase-exit fragment** (ADR-0047),
all **verify-before-return** (each re-decides the three Craig conditions and
declines to `None`/unsupported on any non-`Unsat`/doubt ‚Äî a partial generator can
only over-decline, never emit a wrong interpolant). All ledgered `Validated` (no
per-query certificate emitted; not Lean-reconstructed). **Only the SMT-LIB
`(get-interpolant)` command surface remains** (coordinate `axeyum-smtlib`).

- **T3.8.1 LRA ‚Äî DONE.** `axeyum_solver::lra_interpolant` reads the interpolant off
  the self-checked Farkas certificate (`FarkasCertificate` gained a `vars` field);
  A-only variables cancel automatically, so the shared-vocabulary condition is free.
- **T3.8.3 EUF ‚Äî DONE.** `axeyum_solver::qf_uf_interpolant` summarizes the
  congruence-closure `explain_steps` path, coloring edges by partition and lowering
  non-shared congruence boundaries to argument equalities.
- **T3.8.2 propositional/SAT ‚Äî DONE.** `axeyum_cnf::propositional_interpolant`:
  McMillan partial-interpolant fold over the **LRAT antecedent-hint chains**
  (`axeyum_cnf::lrat`), not the Alethe `resolution` rule (entailment-checked,
  discards pivots). Each `LratStep::Add{clause, hints}` is a RUP chain; replaying
  the forward propagation recovers the pivot per hint ‚Üí a resolution derivation to
  fold over. Input clause id `i ‚â§ |A|` ‚áí A-side. Returns a `BoolExpr` over the
  global (shared) variables, re-verified by SAT (`solve_with_drat_proof` +
  `check_drat`).
- **T3.8.2b QF_BV ‚Äî DONE.** `axeyum_solver::qf_bv_interpolant`: **single joint
  lowering** ‚Äî `lower_terms(arena, A ++ B)` then one `tseitin_encode` (AIG
  structural hashing + per-`TermId` memoization collapse a shared term/symbol bit
  to one `AigLit` ‚áí one `CnfVar`; separate lowerings desync). The single CNF's
  *clauses* are partitioned by per-root reachability (`reachable_node_mask`, now
  `pub`): a clause goes to side S iff one of its vars' AIG nodes is reachable from
  S's roots; a clause reachable from both sides goes into both (sound; makes a
  genuinely-shared bit a *global* var). `propositional_interpolant` over the shared
  space, then each global `CnfVar` ‚Üí `AigLit` (`variable_bindings`) ‚Üí `(TermId, bit)`
  ‚Üí the predicate `((_ extract i i) t) = #b1`. Verify-guarded by the QF_BV decider.
- **T3.8.4 UFLRA ‚Äî DONE.** `axeyum_solver::uflra_interpolant`: one shared
  `eliminate_functions` over A‚à™B, `lra_interpolant` on the function-free
  `abstraction()` (a relaxation ‚Äî unsat there ‚üπ original unsat), fresh vars
  translated back to UF terms. Verify-guarded by `check_with_uf_arithmetic`;
  declines on a congruence-needed (disjunctive) refutation.
- **T3.8.5 fa√ßade ‚Äî DONE.** `Solver::interpolant(arena, a_indices)` dispatches
  LRA ‚Üí EUF ‚Üí UFLRA ‚Üí BV.

## Phase exit criteria
- `(get-interpolant)` works for QF_LRA, QF_BV, QF_UF, and conjunctive UFLRA, with
  **every** returned interpolant verified by independent entailment/`unsat`
  checks before return.
- The CHC engine (P4.6) consumes the interpolation API for frame generalization.
- No unverified interpolant ever escapes; unsupported fragments return a clean
  unsupported/`unknown`, never a guess.
