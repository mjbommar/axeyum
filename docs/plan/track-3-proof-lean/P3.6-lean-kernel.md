# P3.6 — In-tree Rust Lean kernel (`axeyum-lean-kernel`)

**Size:** L · **Depends on:** — (independent; capstone prerequisite) ·
**Frontier**

## Goal
Vendor/reimplement a nanoda-style **pure-Rust Lean type-checker** as
`axeyum-lean-kernel` — a Lean-grade trusted checker with no Lean toolchain and no
C++. This gives axeyum the exact CIC acceptance criterion and a ~6k-LoC reference
checker, the trusted root for the Lean capstone.

## Why / leverage
"Lean accepts the proof" = a CIC `expr` term whose type is the goal, accepted by
the kernel's definitional-equality check. nanoda_lib already does this in pure
Rust (~6k LoC: `tc.rs`, `inductive.rs`, `expr.rs`, `level.rs`, `quot.rs`,
`env.rs`, `parser.rs`), reading Lean's export format. It fits axeyum's no-C/C++
rule; the rework is from `'a`-lifetime arenas to axeyum's `Copy` handles.
