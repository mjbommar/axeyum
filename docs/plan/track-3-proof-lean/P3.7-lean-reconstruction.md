# P3.7 — Alethe→Lean reconstruction (the capstone)

**Size:** L · **Depends on:** [P3.3/P3.5 Alethe proofs](P3.3-alethe-qfbv.md),
[P3.6 Lean kernel](P3.6-lean-kernel.md) · **Capstone — the genuinely hard,
multi-month piece. Do not start until P3.2–P3.5 are solid.**

## Goal
Turn axeyum's Alethe proofs into CIC `expr` proof terms that
[`axeyum-lean-kernel`](P3.6-lean-kernel.md) (and Lean itself) accept — making
axeyum a proof-producing solver usable as a Lean tactic backend. This is "Lean
parity."

## Why / leverage
Lean has no proof *format* — you must *construct* a term whose type is the goal.
The reusable playbook is lean-smt's per-rule lemma library and dispatch pattern
(`@[smt_proof_reconstruct]` handlers, each Alethe/cvc5 rule → a pre-proved Lean
lemma; resolution → `resolution_thm` variants; BV bit-blast → `simp`+`BitVec`
lemmas). Note lean-smt itself walks cvc5's in-memory FFI proof, **not** a text
format — so axeyum builds a *new* Alethe→Lean reconstruction, using lean-smt's
lemma library as the guide.

## Progress notes
- 2026-06-27: `prove_unsat_to_lean_module` and the SOS Lean-module helper now
  retry with a normalized assertion spine when direct reconstruction declines:
  top-level `and` trees are split into fact slices and repeated top-level double
  negations are stripped. This makes consumer-style `hyps ∧ ¬goal` assertions
  reconstruct without caller-side pre-massaging while preserving existing direct
  routes and leaving arbitrary nested Boolean terms untouched. Real-Lean
  regressions cover a normalized QF_UFBV/finite-BV+UF refutation and a
  normalized array read-over-write axiom refutation.
- 2026-07-09 (ADR-0075): the direct equal-array/select-congruence Alethe path now
  uses the shared reconstruction vocabulary (`eq_reflexive`, `eq_congruent`,
  `symm`, resolution) and literal SMT-LIB `select`. The 67-family representative
  gate checks the reversed-read artifact in the real Lean binary; the same proof
  is also accepted by Carcara. No new reconstruction axiom or trust fallback was
  added.

## Phase exit criteria
- A QF_BV `unsat` from axeyum reconstructs into a CIC proof term that
  `axeyum-lean-kernel` (and Lean's kernel) accepts — the first end-to-end
  Lean-checkable SMT proof from axeyum. Coverage then extends rule-by-rule toward
  the certified reductions (P3.5).
