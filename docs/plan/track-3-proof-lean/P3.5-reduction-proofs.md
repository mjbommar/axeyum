# P3.5 — Alethe for the reductions (retire trust-ledger entries)

**Size:** M per theory · **Depends on:** [P3.2 Alethe IR](P3.2-alethe-ir.md),
[P3.0 ledger](P3.0-trust-ledger.md), and the lazy/checkable reductions from
Track 2 · **Front:** reduction certificates (load-bearing #2)

## Goal
Emit Alethe proof steps for the non-bit-blast reductions so each becomes
independently checkable, retiring its [trust-ledger](P3.0-trust-ledger.md) entry.
Priority order follows cvc5's maturity: **arrays (fully checked) → Ackermann
(congruence) → int-blast (rewrites)**, keeping honest holes for fpa2bv and the
int-blast core (matching cvc5's own boundary).

## Progress notes

- 2026-07-09 (ADR-0075): direct `a = b` plus
  `select(a,i) != select(b,i)` now emits literal SMT-LIB `select` and proves the
  contradiction with `eq_reflexive`, `eq_congruent`, optional `symm`, and
  resolution. Forward/reversed artifacts pass the in-tree checker and Carcara;
  removal of the array-equality antecedent is rejected; `produce_evidence`
  carries no trust step; and the representative real-Lean gate checks the
  reconstructed reversed shape. This is select congruence, not the
  disequality-to-diff-witness extensionality rule. Full ROW axioms, diff-witness
  proof production, equality-chain portability, and canonical `CdclT` proof
  logging remain.

## Phase exit criteria
- Arrays, Ackermann, and int-blast reductions emit Alethe steps that Carcara
  (P3.3) / the in-tree checker (P3.4) validate; their trust-ledger entries are
  retired or reduced to a precise residual seam; the aggregate trusted base
  measurably shrinks (golden-tested report).
