# P3.4 — Embedded Alethe checker subset (self-checking)

**Size:** M · **Depends on:** [P3.3 Alethe QF_BV](P3.3-alethe-qfbv.md)

## Goal
Reimplement (or vendor) a checking subset of Carcara's rules in-tree so axeyum
**self-checks** its own QF_BV Alethe without an external dependency — extending
"trusted small checking" above the clausal layer (today only DRAT/Farkas/replay
are in-tree).

## Why / leverage
Carcara is a great external gate (P3.3 CI), but the project identity is *in-tree*
small checkers. A focused subset — exactly the rules axeyum emits — keeps the
trusted base small and self-contained, and is the natural sibling of
`check_drat`/`check_lrat` and `FarkasCertificate::verify`.
