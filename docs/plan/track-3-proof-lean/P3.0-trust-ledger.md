# P3.0 — Reduction trust ledger

**Size:** S · **Depends on:** — · **Do first** (cheap, high-leverage, unblocks the
honest framing of the whole track)

## Goal
Make every trusted reduction a **named, countable** entry with a pedantic level,
mirroring cvc5's `TrustId`. Turn the implicit "modulo trusted reduction" caveat
into an auditable list that P3.5 drives to zero.

## Why / leverage
Today the trust surface is prose. cvc5's central pattern is: a reduction is
"checkable" iff a small self-contained checker re-derives its conclusion;
otherwise it is a typed `TrustId` hole graded 0 (hard fail) … 10 (minor). This
makes the trusted base enumerable — the precondition for shrinking it.
