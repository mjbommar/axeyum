# P1.7 — Propagation-based local-search BV engine (portfolio)

**Size:** L · **Depends on:** — (independent; needs a `BitVectorDomain`-style
constant-bit type) · **ADR required** (new capability surface)

## Goal
Port bitwuzla's propagation-based local search (PBLS) as a second BV engine that
runs as a portfolio partner to bit-blasting — excellent on hard **satisfiable**
QF_BV that bit-blasting chokes on, with no SAT solver involved.

## Why / leverage
The top *new-capability* candidate after the inprocessing/preprocessing wins.
PBLS solves a class of instances bit-blasting is bad at, and reuses a constant-bit
domain shared with the blaster (`PREPROP`-style: run prop with a small budget,
fall back to bit-blasting on unknown).

## Design (port `bitwuzla/src/lib/ls/`)
- Mirror the word-level term DAG as local-search nodes, each with a current
  assignment and a `BitVectorDomain` (fixed/const bits, optionally seeded from a
  bit-blast pass).
- Assertions are roots with target `true`; evaluate bottom-up.
- Main loop: pick a random unsatisfied root; propagate the target value **down one
  path** toward a leaf, preferring an **essential input** (prob 0.99) else random.
- Value selection: if invertible, with prob 0.99 the **inverse value** (exact
  child value making the parent equal the target); else a **consistent value**;
  else conflict, retry. Recompute the affected cone.
- Bound by `max_nprops`/`max_nupdates`; completeness via randomization.

## Phase exit criteria
- PBLS engine decides a set of satisfiable QF_BV instances the bit-blast path
  cannot within budget, each model **replay-checked** by the ground evaluator.
- Integrated as an opt-in portfolio strategy; never returns a wrong answer (LS can
  only conclude sat with a witnessed model; unsat stays with bit-blast/DRAT).
