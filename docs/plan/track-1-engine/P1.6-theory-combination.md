# P1.6 — Theory combination (Nelson–Oppen / interface equalities)

**Size:** M · **Depends on:** [P1.4 e-graph](P1.4-egraph.md),
[P1.5 CDCL(T)](P1.5-cdcl-t-loop.md)

## Goal
Let multiple theories share equalities so combined logics (QF_AUFLIA, QF_AUFBV,
mixed BV+arith) are decided properly rather than as stacked eager reductions.
Adopt Z3's **model-based combination**: identify shared terms, propose interface
equalities between shared terms of equal value, case-split on them.

## Why / leverage
axeyum's current composition is reduction-stacked and eager, with coupling only
propositional — there is no interface-equality propagation. This is why the
BV+LIA gap leaves `str.len` unsat as `unknown`. Combination is required before
multi-theory queries work at parity.
