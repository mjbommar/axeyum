# P1.6 ŌĆö Theory combination (NelsonŌĆōOppen / interface equalities)

**Size:** M ┬Ę **Depends on:** [P1.4 e-graph](P1.4-egraph.md),
[P1.5 CDCL(T)](P1.5-cdcl-t-loop.md)

## Goal
Let multiple theories share equalities so combined logics (QF_AUFLIA, QF_AUFBV,
mixed BV+arith) are decided properly rather than as stacked eager reductions.
Adopt Z3's **model-based combination**: identify shared terms, propose interface
equalities between shared terms of equal value, case-split on them.

## Why / leverage
Before P1.6, axeyum's composition was reduction-stacked and eager, with coupling
only propositional and no interface-equality propagation. ADR-0052 already
closed the documented bounded-string `str.len` UNSAT marker through the exact
linear `bv2nat` bridge, so T1.6.4 is no longer an open combination prerequisite.
The remaining work is broader shared equality and model exchange.

## Current implementation (2026-07-10)

EUF+LIA and EUF+LRA have live backtrackable combined theories with registered
interface `eq`/`lt`/`gt` variables, structural totality/exclusion clauses,
asserted-only conflict explanations, and replay-checked leaf reconstruction.
Boolean-structured QF_UFLIA/QF_UFLRA now run those theories through canonical
`cdclt::CdclT`; the direct conjunctive Nelson-Oppen search remains the model and
replay oracle plus conservative fallback. BV and arrays now have bounded
live-bus slices below; the open phase boundary is deeper array/extensionality/
model participation and opaque-heavy arithmetic model lifting, not another
UF+arithmetic Boolean driver or the already-closed `str.len` marker.

The lazy UFBV/UFLIA functional-consistency routes start from
`abstract_functions`, a projection-preserving relaxation that rewrites each
application once and does **not** materialize any pairwise Ackermann constraints.
This removes the hidden eager O(applications┬▓) construction from the supposedly
lazy paths while retaining identical on-demand lemmas, model projection, and
original-query replay.

The first canonical EUF+BV slice is now live (ADR-0066). Bounded scalar QF_UFBV
builds explicit argument and result equality atoms for same-function application
pairs, then drives `EufTheory` and a warm `IncrementalBvSolver` through one
`cdclt::CdclT` trail. Congruence propagates result equalities into exact BV
semantics; BV-infeasible argument/result arrangements become learned theory
clauses. `Sat` still requires function-table projection and original-query
replay. Construction/search caps return first-class `Unknown`, and the front
door falls through on logical incompleteness. Eager `eliminate_functions`
remains the conservative fallback, differential oracle, and proof-producing
route.

The 512-case direct/eager, front-door/eager, and direct/Z3 matrices are clean,
and the public curated QF_UFBV corpus decides and agrees on 6/6 with zero replay
failures. This is not yet a speed win: the six-row mean is 0.061 s and `bug520`

## 2026-07-02 ŌĆö NAS `hash_sat_*` QF_UFLIA convergence: measured relocation + next slices

Target: the four NAS QF_UF `cli__regress2__hash_sat_{06_19,07_17,09_09,10_09}`
instances (declared `QF_UFLIA`, `:status sat`). Each declares 9 uninterpreted
`hash_k : IntŌåÆInt` and 7ŌĆō9 integer `xßĄó` with `0 Ōēż xßĄó < N`, all `xßĄó` distinct, and
for every `hash_k` the 9 values `hash_k(xŌéü..xŌéē)` are pairwise distinct **and** each
equals some `xŌ▒╝` (i.e. each `hash_k` restricted to `{xŌéü..xŌéē}` is a **bijection** ŌĆö
nine independent permutation/all-different subproblems), plus one nested-`hash`
equality. STATUS localized these as "the online interface-equality loop grinds 248
LP rounds without a sat candidate." Measurement (release, `check_auto_explained`
route trace + targeted probes) **relocates** that:

1. **The grind is the *offline* lazy-arith CEGAR, not the online interface loop.**
   `check_auto` routes these to `uf-arith-lazy-overbound`
   (`euf.rs::check_with_function_consistency` ŌåÆ inner
   `dpll_lia::IncrementalArithDpll::solve`). The outer function-consistency loop runs
   its inner LIA solve **once** (`solve_rounds=1`); that single inner solve grinds
   the DPLL(T) blocking-lemma loop ŌĆö **633 rounds/10 s, 3245 rounds/60 s,
   `sat_candidates=0`, every core `core_src_lp`** ŌĆö and never finds even one
   theory-feasible arrangement of the Ackermannized all-different abstraction. With
   **no theory propagation**, the SAT skeleton re-proposes order-cycles (from the
   `distinct`/`ŌēĀ` ŌåÆ order-disjunction encoding) and the loop blocks one short Farkas
   cycle at a time; the arrangement space is effectively non-convergent.

2. **The online path ŌĆö the one STATUS named ŌĆö never runs under the default caps.**
   `check_qf_uflia_online` declines *instantly* at `MAX_BOOLEAN_ATOMS = 512`

3. **The online path is the right lever (proven).** Raising the caps
   experimentally (`MAX_BOOLEAN_ATOMSŌåÆ8192`, `MAX_OPAQUE_BOOLEAN_ATOMSŌåÆ4096`) lets

4. **Root cause of the leaf-rebuild grind (the actual interface-equality search).**
   At a satisfying CDCL(T) leaf, `cdclt_combined` (`uflia_online.rs` ~L1586)
   **discards the interface arrangement the online `CombinedIncrementalLia` already

   ~1226 literals. That DFS is exponential in the interface pairs; it returns
   `"interface distinct branch inconclusive"` after consuming the entire budget on a

5. **`build_model` note.** The prior build_model fixes (`bff67679`, `86998ea8`)

## 2026-07-02 ŌĆö Slice A implemented, measured, and **relocated**: the blocker is model reconstruction, not the DFS re-search

Slice A (seed the CDCL(T) leaf rebuild with the committed interface arrangement) was
implemented end-to-end and measured against all four
`cli__regress2__hash_sat_{06_19,07_17,09_09,10_09}` (source:
`references/cvc5/test/regress/cli/regress2/`). It **works mechanically but does not
flip any instance**, because the previous diagnosis mis-attributed the grind. The
seeding was reverted; the corrected root cause and the real next lever are below.

### What was built (and reverted)
- An `ArrangementSeed = HashMap<(TermId,TermId), InterfaceRel{Eq,Lt,Gt}>` read off the
  committed leaf: for each `CombinedIncrementalLia::interface_pairs()` pair, read
  `solver.value_of(eq_var/lt_var/gt_var)` (the totality+exclusion structural clauses pin
  exactly one true at a full leaf) ŌåÆ the committed relation, keyed by the pair's `(s,t)`.
- `decide_conjunction_with_seed` threads the seed into the interface DFS (`Search`); at a
  seeded pair the DFS takes the committed branch. Replay gate unchanged (`sat` only).

### Measured results (caps temporarily raised to route these to the online path;
`MAX_BOOLEAN_ATOMS 512ŌåÆ8192`, `MAX_OPAQUE_BOOLEAN_ATOMS 128ŌåÆ4096` ŌĆö required, else
`check_qf_uflia_online` declines instantly; single-file probe, full budget, 10 s)

| instance | seeded (this slice) | unseeded (baseline) |
|---|---|---|
| `hash_sat_06_19` | **unknown, 1.34 s** (walks to committed leaf, replay fails) | **sat, 8.13 s** |
| `hash_sat_07_17` | unknown, 2.28 s (committed leaf, replay fails) | timeout (>10 s) |
| `hash_sat_09_09` | unknown, 3.96 s (committed leaf, replay fails) | timeout (>10 s) |

At production caps (512) all four **decline the online path instantly** and route to the
offline lazy-arith grind ŌĆö unchanged.

### Corrected root cause (supersedes points 4ŌĆō5 above)
1. **Pair alignment is a non-issue.** `combined.interface_pairs()` and the offline
   `interface_pairs(interface_terms(part(leaf-literals)))` are **identical** (measured:
   `06_19` 27=27, `07_17` 35=35, zero missing / zero extra). The interface terms are
   *atomic* ints (`x1..x6`, constants) ŌĆö the `hash_k(...)` apps are not interface pairs;
   congruence over the apps is EUF, not the arrangement. So the seed covers every DFS pair.
2. **The grind is NOT redundant DFS re-search.** The seeded walk reaches the committed
   leaf in ~1ŌĆō4 s (vs the unseeded 8 s search), so the DFS *is* pruned ŌĆö but the committed
   leaf **fails replay on exactly 1 of 997 literals**: the nested-hash equality
   `(= (hash_1 (hash_1 (hash_19 (ite (< (+ x1 x6) 7) (+ x1 x6) x1)))) (hash_1 (hash_1 (hash_19 (ite (< (+ x1 x2) 7) (+ x1 x2) x1)))))`.
   The unseeded DFS visits **321 leaves that all fail this same replay**, then the 322nd
   (`06_19`, 8.13 s) *happens* to pass ŌĆö the search is not exploring arrangements
   inconclusively, it is hunting for a leaf whose reconstruction is faithful.
3. **Why reconstruction is arrangement-fragile.** `combined_model` rebuilds a **fresh**
   LIA witness via `LiaTheory::integer_model()` (re-runs `check_with_lia_simplex` over the
   live conjunction). That witness is **blind to the UF-composition constraint** the
   nested equality imposes, because the online `LiaTheory` treats `hash_k(...)` apps as
   *opaque* integer variables (`new_with_opaque_apps`). It freely picks `x`-values
   satisfying the LIA constraints (`0ŌēżxßĄó<7`, distinct, the arrangement); for most choices
   the two `ite` branches (`(+ x1 x6)` vs `(+ x1 x2)`) select different values, so the
   deep `hashŌłśhashŌłśhash` compositions diverge and the asserted equality fails under the
   ground evaluator. The one-in-322 success is luck in `x`-selection under a forcing order.
4. **Consequence for Slice A.** Seeding the committed arrangement lands squarely on one of
   the 321 reconstruction-failures ŌĆö it cannot flip these. Worse, *pure* seeding (single
   committed branch, no sibling retry) **regresses the online path's completeness**: it
   never reaches the lucky leaf the unseeded DFS finds, turning `06_19`'s `sat` (at raised
   caps) into `unknown`. A **completeness-preserving fallback** (try committed first, then
   the full DFS) is correct but adds the committed-attempt overhead to the online/eager
   budget balance: `uf_arith_dispatch_differential` `budget_excused` measured
   **fallback = 10 vs baseline = 8 vs pure-seed = 7** (all under machine load; the local
   cap is 4 / CI cap 30 ŌĆö *baseline itself is red at 8 under load*, so the gate could not
   discriminate the change locally). Net: seeding is not a verifiable increment here.

### The real next lever (replaces Slice A)
Model reconstruction must **respect UF composition**, not re-pick a composition-blind LIA
witness. Concretely, one of:
- **(a) Congruence-aware witness selection.** After the arrangement is fixed, choose the
  integer values of the *opaque app terms* (not just the original symbols) consistently
  with congruence, then have `integer_model` honor them ŌĆö so `hash_k(a)=hash_k(b)` whenever
  `aŌēĪb` and the asserted app-equalities hold *by construction* rather than by luck.

### The mechanism, pinned by measurement (supersedes the "arrangement-fragile" framing)
Instrumenting the failing leaf confirmed the single replaying failure (`06_19`: term `2419=2424`,
`07_17`: the `(+ x1 x7)` vs `(+ x1 x2)` nested equality) is caused **entirely by function-table
key collisions**: two applications of the *same* hash whose arguments evaluate to the *same*
concrete integer under the LIA witness but whose **class values differ** ŌĆö a bijection-pinned
value (`hash_k(1)=1`) colliding with a fresh composition-class value (`ŌĆ”=6`) at key `1`.
`IntTable::define` keeps the first binding, so the composition chain then reads the wrong value
and the composition literal fails to replay. Measured: 170 such collisions on the first leaf of
`06_19`.

### The fix (landed)
`build_int_uf_functions` now keys every function table by the **concrete** integer value of each
argument (evaluated *innermost-first* through an `eval_with_memo` memo pre-seeded with the values
already fixed for shallower applications, so a nested `hash(hash(ŌĆ”))` argument resolves to the
value we chose, not the still-incomplete table). Each application takes the value forced, in
priority order, by (1) an existing table entry at that key ŌĆö functionality; (2) its
asserted-equality class value (`class_pins` propagates a bijection/`app=atomic` pin *into* the
composition); (3) a fresh distinct value strictly above every constrained value. A key already
bound to a **different** forced value is a genuine infeasibility of *this* witness ŌåÆ a sound
`None` (ŌåÆ `Unknown`), never a wrong verdict; replay stays the final gate. The build is now
**faithful**: it produces a replaying model iff the committed arrangement+witness admits a
consistent hash completion.

### Why `07_17`/`09_09` still do not flip ŌĆö it is now a **search** problem, not reconstruction
Decoded, the `07_17` nested equality is `g(a) = g(b)` with `g = hash_1 Ōłś hash_1 Ōłś hash_17` and
`a = ite((+x1 x7)<8, +x1 x7, x1)`, `b = ite((+x1 x2)<8, +x1 x2, x1)`. Each `hash_k` is asserted to
be a **permutation of {x1..x7}** (the pairwise-`distinct` + `Ōŗü (=hash_k(xi) xj)` blocks), so when a
composition argument lands on an x-value the chain is a bijection and `g(a)=g(b) Ō¤║ a=b`; when it
lands off the x-set the chain is free and the two sides can be set equal by construction. A leaf
therefore replays iff the committed **arrangement + LIA witness** routes the two chains to a
consistent (convergent) pair. The faithful reconstruction accepts exactly those leaves ŌĆö but it
does **not** make them more frequent, and the interface DFS/CDCL(T) does not reach one for
`07_17`/`09_09` within budget (still unknown at 60 s). The remaining lever is the interface
**search**, not the model build:
- The composition **results** are UF results and hence interface terms, so the case-split wastes
  itself equating free `hash(hash(ŌĆ”))` results with x-values (most such guesses are infeasible ŌĆö
  the `pin-candidate conflict func=hash_1 key=[1] cand=1 pin=2..6` declines are exactly these).
  Not generating interface pairs for **unconstrained** (non-`{x1..x7}`) app results, or
  propagating the permutation/all-different structure, is the next step.
- `10_09` never leaves the boolean layer even at raised caps (larger skeleton).
