# P1.2 — Preprocessing (word-level + AIG, shrink before you solve)

**Size:** L · **Depends on:** [P4.5](../track-4-usecases-frontend/P4.5-benchmarking.md)
· **Front:** performance (load-bearing #1)

## Goal
Shrink the problem before bit-blasting and before CNF: word-level BV rewriting,
top-level equation solving, BV slicing/bounds, subterm-sharing maximization, and
two-level local AIG rewriting at construction. Co-priority with
[P1.1](P1.1-sat-inprocessing.md) — preprocessing lowers the work the SAT core
must do and makes inprocessing gains easier to measure.

## Why / leverage
A bit-blasting solver lives or dies on how small and shared the AIG/CNF is.
bitwuzla's speed is largely word-level rewriting + structural-hashed AIG with
two-level rewriting. Z3's `solve_eqs`/`max_bv_sharing`/`bv_slice` are documented
high-impact stages. This is low-risk (each pass is semantics-preserving and
testable in isolation) and high-ROI.

## Dependencies
- Model-soundness: every elimination must be reversible for model
  reconstruction. Build the **model-reconstruction trail** first (T1.2.1) so all
  later passes stay model-sound. This generalizes axeyum's existing lowering/lift
  maps.

## Phase exit criteria
- The preprocessing pipeline runs before bit-blast, is **model-sound** (every
  `sat` still replays), and **measurably reduces AIG/CNF size and PAR-2** on the
  committed slice (recorded in `bench-results/`).
- Each pass is individually flag-gated and unit-tested for semantics preservation.
- Two-level AIG rewriting is deterministic (determinism is a public promise).
