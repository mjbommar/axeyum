# P1.1 — SAT inprocessing (the single biggest performance win)

**Size:** L · **Depends on:** [P4.5 benchmarking](../track-4-usecases-frontend/P4.5-benchmarking.md)
(measure first) · **Front:** performance (load-bearing #1)

## Goal
Add inprocessing to axeyum's CDCL core (`axeyum-cnf`) so the flood of Tseitin
intermediate variables produced by bit-blasting an AIG is collapsed before/while
solving. This is *the* technique that makes bit-blasted CNF tractable and is the
highest-leverage performance item in the whole plan.

## Why / leverage
Tseitin encoding of an AIG introduces one variable per gate; most are redundant.
CaDiCaL/Kissat spend much of their advantage on inprocessing. Bounded Variable
Elimination (BVE) directly targets exactly the structure axeyum generates.

## Dependencies
- A working measured Z3 head-to-head ([P4.5](../track-4-usecases-frontend/P4.5-benchmarking.md))
  so each technique's impact is quantified on a committed slice.
- DRAT must stay correct: every inprocessing step must emit the matching
  DRAT/RAT step (this couples to [P3.1](../track-3-proof-lean/P3.1-lrat.md)).

Implement under `axeyum-cnf` (the CDCL core). Keep all steps **index-based** (no
`unsafe`). Gate each behind a config flag so the benchmark can isolate impact.

## Phase exit criteria
- BVE + subsumption + glue-tier reduction land and **measurably improve PAR-2**
  on the committed QF_BV slice vs the P4.5 baseline (recorded in `bench-results/`
  and STATUS.md).
- Every inprocessing technique emits correct DRAT/RAT; `check_drat` (and, after
  P3.1, `check_lrat`) still validates every `unsat`.
- No regression in the differential-vs-Z3 agreement (0 disagreements, 0 replay
  failures).
