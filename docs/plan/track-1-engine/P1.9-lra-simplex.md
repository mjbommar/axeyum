# P1.9 · LRA feasibility: Fourier–Motzkin → incremental exact-rational simplex

**Size:** L · **Depends on:** the existing FM LRA layer (`lra.rs`, `lra_online.rs`,
ADR-0015) · **Unblocks:** measured QF_NRA/QF_LRA/QF_LIA decide-rate on the
many-variable frontier; lifts the ceiling of the sign-refutation pass (P2.5).

## Why (measured, 2026-07-01)

Both LRA entry points — `check_with_lra` and `check_with_lra_within` — decide
feasibility by **exact-rational Fourier–Motzkin elimination** (ADR-0015). FM is
worst-case *doubly exponential* in the variable count: each eliminated variable can
square the constraint count, so it hits the `MAX_FM_CONSTRAINTS = 20_000` guard and
declines (`unknown`) on modest many-variable systems.

Measured impact on the QF_NRA `cvc5-regress-clean` set (`explain_corpus`):

- **~8 instances** decline directly with `lra: Fourier–Motzkin elimination exceeded
  the wall-clock / size budget`.
- The **sign-refutation pre-cap pass** (`f9e06baf`) is *capped by the same FM
  budget*: it decides Boolean-*pinned* cases (`simple-mono-unsat`) where a variable
  is fixed per branch, but declines fully-free many-product systems (`metitarski-*`,
  `poly-1025`) because the sign-lemma LRA system overruns FM. An experiment that
  added the (linear) threshold-1 monotonicity lemmas to that pass **regressed** even
  the pinned cases — the extra disjunctive lemmas enlarge the FM system past the
  budget before it refutes. So the bottleneck is *the feasibility engine, not the
  lemmas*.

Simplex decides feasibility in (empirically) polynomial time and, on infeasibility,
its dual yields a **Farkas certificate directly** — which is exactly the artifact
the current proof route (`lra_farkas_certificate`) already consumes.

## Design

A new `crates/axeyum-solver/src/simplex.rs` — an **exact-rational** (`axeyum_ir::Rational`,
overflow → graceful decline) linear-feasibility core:

- **Phase-1 simplex** over a tableau of `Σ aᵢⱼ xⱼ ⋈ bᵢ` (⋈ ∈ {≤, <, =}); a
  bounded/auxiliary formulation decides feasibility. **Bland's rule** for
  anti-cycling guarantees termination (no wall-clock reliance for correctness).
- **Strict inequalities** via the standard `δ`-relaxation (symbolic infinitesimal:
  values are `(rational, δ-coeff)` pairs), so `<`/`>` are exact, not
  epsilon-approximated.
- **Output:** `Feasible(model)` (a rational point — replay-checkable) or
  `Infeasible(farkas)` where `farkas` is `y ≥ 0` with `yᵀA = 0` and `yᵀb < 0`
  (⟨strict rows contribute to a `< 0`⟩) — the same nonnegative-combination
  certificate `lra.rs` already self-checks before returning.
- Overflow in the exact arithmetic → `Unknown` (never a wrong verdict), matching the
  `checked_*` discipline (see [solver Rational overflow class]).

4. **T1.9.4** — once differential-clean at scale, make simplex the default LRA
   feasibility core; FM retained only as a cross-check in tests. Re-measure

### T1.9.2 integration point (located 2026-07-01, ready to execute)

The clean hook is the **`Decision::TimedOut` branch of `check_with_lra_within`**
(`lra.rs`, the `Fourier–Motzkin … exceeded` case): that is exactly "FM blew up",
so before returning the FM-budget `unknown`, translate the same assertions to
`simplex::Constraint` and call `simplex::feasible`:

- **Parsing.** Reuse `lra.rs`'s existing `LinExpr` / `linearize` (the per-atom
  linear-form extractor at `lra.rs:663–727`) — it already yields `Σ cⱼ·xⱼ ⋈ b` with
  a dense variable index. Map its atom `(lhs, strict)` + comparator to
  `simplex::{Constraint, Rel}` (`RealLe→Le`, `RealLt→Lt`, `RealGe→Ge`, `RealGt→Gt`,
  real `=`→`Eq`). Do **not** hand-roll a second parser (it must match FM's semantics
  exactly) — factor `linearize` out or expose it `pub(crate)`.
- **Results.** `Feasible(point)` → `Sat` (rebuild the IR model from the
  index→symbol map; the point is already replay-checkable). `Infeasible(y)` with a
  non-empty cert → `Unsat` (thread `y` into `Decision::UnsatFarkas` so the existing
  `lra_farkas_certificate` self-check + Alethe `la_generic` emitter consume it
  unchanged); an **empty** cert (strict-δ, not yet extracted) → keep the FM
  `unknown` (never an uncertified `unsat`). `Unknown` → keep the FM `unknown`.
- **Gate.** Soundness-critical LRA change → `nra`+`nia`+`lra` differential fuzz
  DISAGREE=0 + `carcara_crosscheck`. Add a size threshold so small systems stay on
  FM (no behavior change there); only FM-timeout cases reach simplex at first.

## Soundness

- Simplex feasibility is an **exact decision** over the rationals (Bland's rule ⇒
  termination; δ-relaxation ⇒ exact strict handling); every `Feasible` model is
  **replay-checked** against the original atoms exactly as today.
- Every `Infeasible` returns a Farkas `y`; it is **re-checked** (`yᵀA = 0`,
  `yᵀb < 0`, `y ≥ 0`) before it is trusted — a failed self-check is a loud alarm,
  never a silent `unsat`. This is the same trust gate FM uses.
- Gate the whole change with the existing differential fuzzes (`nra`/`nia`/`lra`
  vs Z3, DISAGREE=0) and the `carcara_crosscheck` (the `la_generic` certificate
  shape must stay Carcara-valid).

## Exit criteria

- `simplex::feasible` decides the FM-declining QF_NRA/QF_LRA instances; the
  committed scoreboard shows a measured decide-rate gain with **DISAGREE=0**.
- Every `unsat` still carries a self-checked Farkas certificate; `carcara_crosscheck`
  stays green.
- An ADR records the FM→simplex feasibility-core decision, the δ-relaxation for
  strict atoms, and the retained-FM cross-check.
