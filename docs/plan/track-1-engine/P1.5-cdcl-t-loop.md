# P1.5 — CDCL(T) loop (theory integration) — KEYSTONE

**Size:** L · **Depends on:** [P1.4 e-graph](P1.4-egraph.md) · **Keystone:** the
eager→lazy upgrade for the whole theory layer

## Goal
Add the CDCL(T) loop: theories plug into the existing CDCL SAT core as
extensions, get the SAT trail, run `check()` (cheap propagation + expensive
final-check), and feed conflicts/lemmas/propagations back. This converts axeyum's
one-shot reductions into lazy, integrated theory solvers.

## Why / leverage
This is the structural fix the reference review names as the eager→lazy keystone
(alongside the e-graph). It unlocks: lazy BV multiplier blasting, lazy array
axioms, the arithmetic `check_lia`/`check_nla` interleave, and quantifier
instantiation rounds — i.e. most of Track 2's performance.

## Current implementation (2026-07-09)

Canonical `cdclt::CdclT` now drives EUF, strings, pure LIA/LRA first probes, and
the live EUF+LIA/EUF+LRA combined theories. It provides online assertion,
lockstep backtracking, theory propagation with asserted-only explanations,
mixed Boolean/theory 1-UIP learning, deadlines plus a no-deadline step belt,
deterministic conflict-side VSIDS, phase saving, and Luby restarts. Remaining
driver-quality work is LBD-based learned-clause reduction, followed by the P1.6
BV/array theory adapters and genuinely warm frontend reuse.

## Design (port the modern Z3 `sat::extension` architecture)
- A `TheorySolver` trait analogous to `sat::extension`: `propagate`,
  `propagated(lit, idx)`, `get_antecedents(lit, idx) -> Vec<Lit>` (**lazy
  explanation**), `decide`, `final_check() -> {Done, Continue, Giveup}`,
  `asserted(lit)`, `push`/`pop`.
- Theories share the [e-graph](P1.4-egraph.md) (`th_eq` records on class merges).
- Conflict resolution extended so antecedents can be **equality justifications**
  expanded into literals (the e-graph's `explain`).
- `final_check` is the lazy expensive pass; cheap propagation runs eagerly.

## Phase exit criteria
- A working CDCL(T) loop: at least EUF runs as a lazy `TheorySolver` on the
  e-graph, deciding QF_UF without Ackermann eager-elimination, with replay-checked
  models and learned theory conflict clauses.
- Conflict resolution correctly expands equality justifications (verified against
  the eager path on a differential test).
- Push/pop integrates cleanly with the e-graph trail and the warm SAT solver.
