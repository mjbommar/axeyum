# P4.7 — Synthesis & abduction (SyGuS, abducts, optimization-as-synthesis)

**Size:** L · **Depends on:** [P2.6 quantifiers](../track-2-theories/P2.6-quantifiers.md),
[P3.8 interpolation](../track-3-proof-lean/P3.8-interpolation.md),
[P1.5 CDCL(T)](../track-1-engine/P1.5-cdcl-t-loop.md) ·
**Abduction seeded; general SyGuS absent · lower priority**

## Goal
Add the *inverse* of solving — instead of "is there a value," answer "is there a
**function/formula**." Three related cvc5/Z3 features:
1. **SyGuS** — synthesize a function from a grammar satisfying a spec
   (`synth-fun` + constraints), via counterexample-guided inductive synthesis.
2. **Abduction** — given `A` and goal `G` with `A ⇏ G`, find the weakest `C` such
   that `A ∧ C ⇒ G` and `A ∧ C` is consistent (`get-abduct`).
3. **Interpolant-as-synthesis** — reuse P3.8 where the "function" is a predicate.

## Why / leverage
This is breadth that turns axeyum from a *checker* into a *generator*, and it is
exactly what the K-12/curriculum frontend wants ("find the rule, not just the
number") and what invariant-discovery (P4.6) and program repair want. It is
genuinely lower priority than CHC/interpolation — list it, scope it, but it waits
behind the keystones. It leans hard on machinery built elsewhere
(quantifier instantiation, MBP, interpolation), so it is mostly *integration*
once those exist.

## Design notes
- **CEGIS loop:** candidate from the synthesis solver → verify against the spec
  (a `forall` query, decided by P2.6) → on failure, a counterexample refines the
  candidate set. The verify step reuses the quantifier/instantiation engine; the
  candidate step is enumerative or unification-based (cvc5 `sygus_unif`).
- **Grammar:** a `synth-fun` grammar lowered to a datatype of syntax trees
  (the standard SyGuS-to-datatypes encoding) — reuses datatypes (P2.9).
- **Abduction:** dual to interpolation (P3.8); a quantifier-elimination /
  conjecture-then-check loop.
- **Contract:** a synthesized function is **verified against the full spec**
  (the universally-quantified correctness query must be `unsat`-of-negation,
  checked) before it is returned; abducts are checked for both entailment and
  consistency. No unverified synthesis result escapes.

## Current status

Abduction and general synthesis must be reported separately. The public
full-profile `abduct` API already performs bounded shared-vocabulary search,
synthesizes equalities and arithmetic comparisons, and rechecks consistency,
sufficiency, and vocabulary before returning. The
[2026-07-21 categorical-engine audit](../categorical-engine-depth-audit-2026-07-21.md)
records 9/9 focused tests passing under a hard 4 GiB cap.

There is no textual `get-abduct`, weakest/minimality guarantee, user grammar,
general CEGIS loop, `synth-fun` parser, or SyGuS-IF corpus. Thus abduction is a
verified seed while general SyGuS remains absent; neither “all synthesis is
missing” nor “synthesis is done” is accurate.

## Phase exit criteria
- A small committed SyGuS slice is solved with every synthesized function
  **verified against its full specification** before return.
- `get-abduct` returns abducts verified for entailment + consistency.
- `unknown`/giving up is first-class; no unverified function or abduct escapes.
