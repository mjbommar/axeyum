# P4.3 — Optimization: OMT lexicographic/Pareto + MILP hardening

**Size:** M · **Depends on:** — (independent; builds on existing `optimize.rs`,
`maxsat.rs`, MILP) · **Use case:** constrained program optimization

## Goal
Grow axeyum's optimization from single-objective (`maximize_bv`/`minimize_lia`,
MaxSAT, MILP — currently `experimental`) to Z3-`opt`-class: **lexicographic** and
**Pareto** multi-objective optimization, with hardened MILP and per-step certified
optima.

## Why / leverage
"Constrained program optimization" is a named first-class use case. The pieces
exist (per-step optimum certified by the underlying procedure; path-condition
optimization landed in `SymbolicExecutor`); what's missing is multi-objective
orchestration and robustness.

## Current slice
Lexicographic, box, and Pareto OMT modes exist for the main BV/LIA surface, and
the consumer-facing counterexample path now has a shared minimization helper:
`minimize_model` / `minimize_model_with_config` and `Solver::minimize_model`
lexicographically minimize a replay-checked satisfying model over caller-selected
Bool, unsigned-BV (`w <= 127`), and Int symbols. `produce_evidence_minimized`
and `prove_minimized` surface the same strict minimizer through the evidence and
proof front doors, so property and verification frontends can request a
deterministic "small failing input" without reimplementing optimizer loops
client-side. Wider BV, Real, array, datatype, and uninterpreted-carrier
objectives remain explicit unsupported/unknown cases rather than silently
producing a non-minimal model.

2026-06-27 SDK slice: `axeyum-property` v0 is the first typed consumer of this
proof/minimization surface. It provides `Bool`, `Bv<W>`, and `Int` handles over
`TermArena`, records declared scalar symbols in deterministic objective order,
delegates to `prove` / `prove_minimized`, lifts scalar model values through typed
handles, and exposes reusable typed unsigned BV overflow predicates for EVM and
Rust-verifier frontends. It is deliberately a thin SDK; no solver logic lives in
the crate.
