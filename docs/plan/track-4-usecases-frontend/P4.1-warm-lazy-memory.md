# P4.1 — Warm lazy arrays / symbolic memory

**Size:** L · **Depends on:** [P1.4 e-graph](../track-1-engine/P1.4-egraph.md),
[P1.5 CDCL(T)](../track-1-engine/P1.5-cdcl-t-loop.md),
[P2.2 lazy arrays](../track-2-theories/P2.2-arrays-lazy.md) · the deferred half of
ADR-0030

## Goal
Make the incremental engine handle arrays and memory-shaped UFs **warmly** so
memory-bearing symbolic execution / BMC / k-induction stop re-eliminating them
eagerly on every `check`/depth. Today the warm path accepts only narrow
array/UF slices; `check_with_memory` still re-eliminates general deferred theory
queries per call (one-shot), which is the perf wall for memory workloads.

## Why / leverage
The already-shipped `bounded_model_check_with_memory` and `check_with_memory`
re-solve from scratch each depth. With lazy arrays on the warm CDCL(T) engine
(P2.2), learned clauses and array axioms persist across depths — the difference
between toy and real symbolic-execution-over-memory.

Interim U7 mitigation (2026-06-27): `SymbolicMemory` now has a frontend
write-log helper that drops later-shadowed same-index writes and emits a
read-specific compact read-over-write `ite` chain. It skips writes whose literal
address is known distinct from the read, elides exact-hit guards, and still
guards later symbolic writes that may alias. This reduces consumer-side
store-log blowup for simple memory logs, but it is not a replacement for
retained lazy-array/UF theory clauses. The warm ROW simplifier also prunes
earlier same-index stores shadowed by a later store before expanding an
undecided symbolic read, so `select(store(store(a, i, old), i, new), j)` emits
one equality guard for the live write instead of retaining the dead old value in
a nested guard chain. Trivial scalar `ite`s produced by memory simplification
also collapse after their branches simplify, so branch-merged memory reads like
`select(ite(c, store(a,i,v), store(b,i,v)), i)` reduce to `v` rather than
bit-blasting a dead `ite(c, v, v)` merge guard. Reflexive equalities exposed by
those rewrites also collapse (`v = v -> true`, `not true -> false`), so
tautological memory constraints and their negations avoid residual Boolean
encoding work.

First warm admission (2026-06-27): `IncrementalBvSolver` can now assert a
warm-safe simplification of a narrow memory slice while retaining the
original assertion for model replay and memory-aware checks. Same-index
store/read-back terms collapse (`select(store(a, i, v), i) -> v`), and
literal-distinct concrete-address misses skip the unrelated store
(`select(store(a, c1, v), c2) -> select(a, c2)` when `c1 != c2` is known from
constants), recursively through store chains. Reads from constant arrays also
collapse to the default value (`select((as const Array) v, i) -> v`), covering
zero-initialized memory before any symbolic write. Reads over array-valued
`ite`s distribute to scalar branch reads
(`select(ite(c, a, b), i) -> ite(c, select(a, i), select(b, i))`), covering
simple branch-merged memory states when the selected branches reduce through the
same narrow slice. Reads over index-valued `ite`s also distribute
(`select(a, ite(c, i, j)) -> ite(c, select(a, i), select(a, j))`), so
conditional symbolic addresses split before read-over-write handling and can
reuse the same-index, literal-distinct, constant-array, and shadowed-store
rules branch-by-branch. Reads over stores whose write index is an `ite` split
to branch-local store/read terms
(`select(store(a, ite(c, i, j), v), k) -> ite(c, select(store(a, i, v), k), select(store(a, j, v), k))`),
so conditional writes get the same branch-local simplification before the
generic symbolic ROW equality. Symbolic-address read-over-write also expands to
the total SMT-LIB conditional
(`select(store(a, i, v), j) -> ite(i = j, v, select(a, j))`) and then keeps
simplifying the else branch, so symbolic hits/misses over zero-initialized or
otherwise reducible store chains can stay warm. Store chains are first pruned
for syntactically shadowed same-index writes under the later store, avoiding
redundant old values and duplicate guards in simple write-log shapes. The same
simplifier collapses trivial scalar `ite`s exposed by these rewrites, which
removes branch-merge guards when both selected array branches read back the same
scalar value, and then folds reflexive equalities / Boolean-constant negation
exposed by the scalar result. It also distributes equality over Bool/BV-valued
`ite`s and folds literal-distinct constant equalities plus Boolean identity
`ite`s, so branch reads such as `ite(flag, 1, 0) = 1` collapse to the path
condition before warm encoding. Boolean equality-to-constant cleanup and
double-negation removal also make symbolic Bool readbacks such as
`select(store(mem, i, p), i) = true` collapse to `p`; Boolean connective
cleanup also removes constant, idempotent, and complement wrappers around
predicate-like readbacks, so `select(store(mem, i, p), i) and true` collapses to
`p` and `p and not p`-shaped readbacks collapse to `false`. Boolean
xor/implication cleanup covers the same predicate-like readbacks:
`p xor false -> p`, `p xor p -> false`, `true => p -> p`, and
`p => false -> not p`. BV bitwise cleanup removes common mask wrappers around
BV-valued readbacks too: `x & all_ones -> x`, `x | 0 -> x`, `x xor x -> 0`,
and double `bvnot` collapses after the readback reduces. BV arithmetic cleanup
then removes exact modular zero/one wrappers around readbacks:
`x + 0 -> x`, `x - 0 -> x`, `x - x -> 0`, `0 - x -> bvneg x`,
`x + bvneg x -> 0`, `x * 1 -> x`, `x * 0 -> 0`, and double `bvneg`
collapses. BV div/rem cleanup admits exact SMT-LIB-total cases too: unsigned
division by zero collapses to all ones, division by one collapses to the
source, zero divided by a syntactically nonzero divisor collapses to zero,
remainder/modulo by zero or one collapse to the corresponding total result, and
self-remainder/self-modulo collapses to zero. Signed division by zero and
nontrivial variable divisors remain ordinary BV terms. BV comparison cleanup
removes reflexive and unsigned endpoint wrappers around readbacks too:
`x < x -> false`, `x <= x -> true`, `x <u 0 -> false`,
`0 <=u x -> true`, `x <=u all_ones -> true`, and
`x >=u 0 -> true`. BV slice/extension cleanup removes whole-width extracts and
zero-bit `zero_extend` / `sign_extend` wrappers around readbacks, trimming
frontend bitfield and width-normalization scaffolding while leaving partial
extracts and positive-width extensions as ordinary BV terms. BV shift cleanup
removes exact no-op/constant-result wrappers too: `x << 0`, `x >>u 0`, and
`x >>s 0` collapse to `x`, zero shifted by any amount collapses to zero, and
all-ones arithmetic-right-shifted by any amount stays all ones; nonzero
variable shifts and over-shift rewrites remain ordinary BV terms. It covers
one-shot assumptions and assumption cores: the warm solver encodes the
simplified BV term but replays models and reports UNSAT cores against the
original memory assumption. `SymbolicExecutor::assume_auto` and `branch` use
this route, so
simple store/read-back memory path constraints, concrete-address miss chains,
zero-initialized reads, simple array-ITE state merges, reducible conditional
read/write-index paths, reducible symbolic-address ROW, and fork queries can
stay on ordinary warm `check`/`model`. This is still not general lazy arrays:
symbolic base-array state beyond the retained BV-indexed Bool/BV select slice,
general extensionality over arbitrary array terms, stores that survive as array
values, non-scalar array component sorts, and general UF lemmas remain deferred
to the one-shot dispatcher.

First retained warm array-read abstraction (2026-06-27): after the syntactic
memory simplifier runs, `IncrementalBvSolver` can keep `select(a, i)` warm when
`a` is a BV-indexed array symbol whose element is Bool or BitVec and `i` is
array-free. Each read is abstracted to a retained internal warm scalar variable,
same-array select pairs get selector-scoped congruence lemmas
`i != j || select(a,i) = select(a,j)`, and SAT models are projected back into
concrete array entries before replaying the original assertion or one-shot
assumption. This closes the common symbolic-base load case without invoking
`check_with_memory`, including helper-level load conditions, Bool-valued
predicate/set reads, and symbolic-address ROW tails whose remaining base read is
a plain array-variable select. BV-array projection keeps the compact
`ArrayValue` model path only when both the index and element widths fit in
`u128`; wide index or element values, including BV256 storage-style reads, are
projected through `GenericArrayValue` so replay sees the full wide-BV value. It
is an Ackermann/congruence warm slice, not yet the full lazy array engine.
Direct equality between supported array symbols now joins that retained slice:
the equality is kept as a scoped warm theory fact, equal-array classes generate
cross-array select-congruence lemmas for committed assertions and one-shot
assumptions, and SAT models merge equal arrays before replay. This covers the
common `a = b`, `i = j`, `select(a,i) = x`, `select(b,j) = y` conflict shape
without invoking the dispatcher. ROW/extensionality instantiation for arbitrary
array terms, disequality witnesses, learned theory-clause retention for the
general array engine, and broader UF/array combination still remain.

First retained warm scalar-UF abstraction (2026-06-27): the same abstraction
machinery now covers scalar Bool/BV applications `f(args)` when all argument and
result sorts are Bool or BitVec, including BV256 keccak-style values. Each
application is replaced by a retained internal warm variable; same-function
applications receive selector-scoped congruence lemmas
`(args differ) || value_i = value_j`; and SAT models are projected back into
`FuncValue` entries before original-term replay. Compact Bool/BV<=128
signatures still use the scalar function table; wide-BV argument or result
signatures use full-value `FuncValue` entries so replay sees canonical
`Value::WideBv`s. This closes the keccak-style scalar UF branch case for
committed assertions and one-shot assumptions without invoking
`check_with_memory`. It is still not full EUF on the warm path:
Int/Real/array/datatype/uninterpreted-carrier arguments/results, quantifiers,
extensionality, and learned theory-clause reuse across the general array/UF
engine remain open.

ADR-0086 adds the first retained structural-read semantics. The solver's
admission pass now preserves undecided reads over store, constant-array, and
array-ITE parents, gives each a private scalar owner, and installs its exact ROW/
constant/ITE definition once in the persistent CNF. Generated base/branch reads
recurse through the existing leaf/UF abstraction; only direct symbol leaves
project array models, and original replay remains mandatory. User roots and
array-equality-dependent cross-symbol consequences stay selector-scoped, while
private definitions safely survive pop and opposite one-shot branch checks.
Exact 512-node and 256-depth limits defer one-over roots. The 192 warm/
`check_auto`/Z3 comparisons, full solver/symexec/EVM gates, and DISAGREE=0 EVM
scoreboard are clean. This is observation-triggered, not yet candidate-triggered:
the depth-32 EVM row measures 30.933 ms warm vs 0.368 ms frontend ITE-fold, so
the performance exit remains open and ITE-fold remains the consumer default.

ADR-0087 makes structural ROW candidate-triggered and replaces the measured-
regressive one-step prototype with one exact transitive scalar summary per
observed read. Summaries stay as dormant IR metadata until a candidate violates
them, then become permanent roots in the same incremental CNF under one shared
deadline. A replayable miss can install zero definitions; a nested violated
store chain activates one compact summary. Scope/core/reuse gates and the 192
warm/`check_auto`/Z3 comparisons remain clean. Release depth 32 improves 2.75x,
from 30.933 ms to 11.257 ms, but ITE-fold remains faster at 0.405 ms and stays
the default.

ADR-0088 makes scalar-keyed array-valued UF applications retained warm leaves.
Private application arrays carry observed reads through conditional argument/
index congruence; equal concrete tuples merge split observations into one full-
value function result before private owners are filtered and originals replay.
The exact 64/65-parent boundary, ten focused tests, and 192 warm/`check_auto`/Z3
comparisons pass. The EVM corpus has no array-valued UF application, so this is a
capability increment rather than an EVM timing change.

ADR-0089 retains top-level array relations. Positive equality merges direct or
application projection owners before function construction. Disequality over
any supported symbol/store/constant/ITE/application parent gets one private diff
index and two retained reads, so candidate-triggered ROW supplies exact witness
semantics. The 192 relation comparisons pass; current EVM paths do not construct
whole-array relation roots, so there is no timing delta.
