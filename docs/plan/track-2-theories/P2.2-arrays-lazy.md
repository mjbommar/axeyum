# P2.2 — Arrays: lazy axiom instantiation

**Size:** L · **Depends on:** [P1.4 e-graph](../track-1-engine/P1.4-egraph.md),
[P1.5 CDCL(T)](../track-1-engine/P1.5-cdcl-t-loop.md) · **The biggest array
scalability win**

## Goal
Move arrays from eager elimination (read-over-write + Ackermann, ADR-0010 — which
blows up on store chains and large index spaces) to **lazy axiom instantiation**
on the e-graph, matching Z3.

## Phase exit criteria
- Lazy arrays decide store-chain / large-index instances that the eager path
  cannot within budget, with replay-checked models; warm path no longer refuses
  arrays (ties into [P4.1](../track-4-usecases-frontend/P4.1-warm-lazy-memory.md)).
- An [Alethe reduction proof](../track-3-proof-lean/P3.5-reduction-proofs.md) for
  read-over-write (`ARRAYS_ROW/IDX/EXT`) becomes feasible (arrays are cvc5's
  fully-checked case).

## Current Boundary (2026-07-10)

ADR-0071 lands abstraction-only base selects and candidate-batched select
congruence. ADR-0072 moves store reasoning out of eager preprocessing on the
canonical ABV/AUFBV route: the shared ROW abstraction records store sites, and
only candidate-violated hit/miss semantics materialize exact guarded axioms.
Function applications in ROW metadata remain shared, and original replay still
gates SAT. ADR-0084 subsequently makes canonical projection array-first then
function-second. A 24-write concrete-miss chain
replays in one round with zero ROW axioms; hit/miss controls each require one.

ADR-0081 makes that local ROW refinement genuinely same-search. Each store site
reserves three bounded atoms; a violated candidate activates them with two valid
permanent clauses and resumes the same `CdclT` instance with learned search state
retained. Hit/miss and nested-store controls now close in one outer round, and a
UF-bearing index can propagate through the already-aligned e-graph atom without
a separate function-pair rebuild.

ADR-0082 extends the same retained search to pair-generated scalar atoms. The
driver maps appended SAT variables to aligned EUF/BV atoms explicitly; EUF
registration is limited to pre-observed equality sides, and exact BV owns the
arena clone that interns the abstract forms. Function pairs, explanation-guarded
parent-select pairs, and bounded equality observations now refine in one
canonical round without preparing cross products. The dynamic-interface matrix
brings the clean total to 2,304 comparisons and the solver library to 809 tests.

ADR-0084/0085 subsequently close finite-scalar array-result ownership and the
bounded structural store/ITE/constant total-model boundary; ADR-0088 carries
scalar-keyed array-result ownership onto the retained warm path. The remaining
work is warm equality/extensionality and broader class-model reuse, nested or
extended array operators, and proof logging. ADR-0072/0081 provide candidate-
guided same-search local ROW, while ADR-0082 provides pair-generated scalar atom
growth; this is not yet the complete Z3-style array-theory queue.

ADR-0073 adds the first canonical extensionality slice. Shared `RowCtx` equality
flags receive one diff witness plus paired reads at already-relevant query/store
indices. Only candidate-violated true-congruence or false-diff implications are
materialized, and UF-bearing observation roots remain on the same bus. ADR-0076
expands the matrix to 456/768 equality-bearing comparisons, all clean. This
completes T2.2.3 only for bounded candidate-guided observations: live class-merge
triggers, parent/alternate-path scheduling, e-graph-class models, warm reuse, and
proof artifacts remain.

ADR-0074 adds the first scalable model slice. The shared projection helper now
votes over distinct observed indices, chooses a deterministic majority else
value, and retains only true overrides for both compact BV and generic arrays.
An end-to-end 16-read canonical model uses one round, default `7`, and four
overrides for 12 repeated and four minority values. All differential/public
decisions and replay remain unchanged. T2.2.4 still retains e-graph-class model
ownership, warm reuse, nested/extended arrays, and proof integration.

ADR-0075 closes one proof-side leaf without changing solver behavior: direct
`a=b` plus same-index unequal reads now emits one literal-`select` artifact
accepted in-tree, by Carcara, and by real Lean with no array-elimination trust
step. This is ordinary select congruence. ROW instances, the disequality
diff-witness direction, equality-chain portability, and canonical online proof
logging remain P3.5 work alongside this phase's queue/class-model depth.

ADR-0076 added the first explicit queue state and cross-atom scheduling slice.
For each candidate-false equality, deterministic BFS over candidate-true flags
either leaves the obligation delayed or applies only the missing path-edge reads
at that equality's own diff index. The transitive `a=b ∧ b=c ∧ a≠c` gap closes
with two observations; disconnected SAT, store/UF+ROW composition, and the exact
512 cap are pinned. This avoids eager quadratic preparation, but the queue still
lived across rebuilt canonical rounds rather than inside a backtrackable array
theory attached to `CdclT`.

ADR-0077 supersedes that compensating queue. The flag remains the exact
BV/Boolean atom while its aligned EUF term is the original array equality, so
ordinary reflexivity/transitivity/congruence now runs on the live backtrackable
e-graph. The former transitive, self, store/UF, and 512-observation cases refute
in one round with no array axiom. Projection unions candidate-true direct-symbol
classes and builds one shared majority-default model, closing a transitive SAT
replay gap with disjoint observations. The 790-test library and strengthened
768-comparison matrix are clean. Parent-select merge scheduling, non-symbol/warm
class models, and proof artifacts remain.

ADR-0078 adds the first actual parent-select merge hook. Every base read parent is
pre-registered with `EufTheory`; a total candidate groups reads by final e-class
and materializes only equal-index/unequal-result pairs. Cross-parent lemmas carry
the e-graph explanation as a guard, so a later Boolean branch can retract the
merge without retaining an invalid unconditional equality. Direct-symbol
equalities no longer prepare every query index: an 80-array/80-read SAT gate now
stays under the 4,096 ROW-site cap. Structural equality observations remain for
store/ITE interaction. All 794 solver tests, 768 comparisons, and public decisions
remain clean. ADR-0080 subsequently adds store parents and ADR-0081 inserts
bounded local ROW inside one search; ADR-0082 subsequently moves pair-generated
scalar select/extensionality atoms into that search. ITE/default/UF parent events
requiring new e-graph terms, warm reuse, non-symbol models, and proofs remain.

ADR-0079 widens only the already-exact finite-scalar boundary: Bool and BitVec
are independently valid index/element components. Mixed arrays use the generic
majority-default model representation and remain original-replay-gated; an Int
negative control confirms that other component theories are not admitted.
`issue5925` and `issue4240` become decisions, and 384 new analytic/front-door/Z3
comparisons are clean. This does not advance structural class ownership or warm
reuse, so the next work remains T2.2.1 and the residual of T2.2.4.

ADR-0080 adds the first structural parent kind to the same scheduler. A store
site now retains its original store term, which is pre-registered with
`EufTheory`; equal-index/unequal-result reads from one final store-parent class
materialize the existing select implication, guarded by the merge explanation
when the parent terms differ. The site simultaneously remains on the lazy ROW
scan, keeping parent congruence and read-over-write independent. Five focused
mechanism/branch/scale gates and 384 structural eager/front-door/Z3 comparisons
pass, bringing the clean total to 1,536 and the solver library to 802 tests. The
remaining T2.2.1 boundary is array-valued ITE/default/UF and merge-triggered ROW
events, pair-generating dynamic insertion, warm reuse, non-symbol models, and
proof logging.

ADR-0081 adds the first dynamic in-search array refinement. Newly created local
ROW atoms are reserved dormant; the candidate final-check inserts two permanent
ROW clauses, activates the site, and resumes the same `CdclT` search. Driver
activation/backtracking tests, one-round hit/miss and nested-site gates, replayed
branch changes, exact-cap behavior, and a UF-bearing index gate pass. The new
384-comparison matrix brings the clean total to 1,920 and the solver library to
807 tests. Pair-generated parent/select/extensionality events, array-valued
ITE/default/UF ownership, warm reuse, non-symbol models, and proof logging remain.

ADR-0082 appends those pair-generated scalar equality atoms inside the live
search. Explicit SAT/theory maps handle variables created after Tseitin
auxiliaries; EUF atoms reuse pre-observed sides; exact BV grows in its owned
arena. Function, base/store-parent select, and bounded extensionality refinements
retain learned clauses, phase/activity state, e-graph state, and warm BV state.
One-round mechanism, backtracking, cap, scale, eager/front-door, and Z3 gates
pass; 2,304 comparisons and 809 solver tests are clean. Array-valued
ITE/default/UF events requiring new e-graph terms, warm reuse, non-symbol models,

ADR-0084 closes the UF-result part of those historical residuals. Online ROW
recognizes `select(f(args), i)`, retains `f(args)` as the semantic e-graph parent,
and maps the read to function abstraction's fresh array projection owner. Final
classes union split observations across congruent applications before
array-first/function-second projection builds full-value function results and

nested scalar-UF use pass six focused gates plus 288 analytic/front-door/Z3

ADR-0085 closes that bounded structural ownership residual. Array-ITE equality

ADR-0086 starts that warm reuse boundary in `IncrementalBvSolver`. Observed

ADR-0087 makes the warm ROW slice candidate-triggered. One exact transitive

ADR-0088 closes the scalar-keyed warm array-valued UF-parent residual. Reads

ADR-0089 closes projection-owned positive equality and exact top-level
