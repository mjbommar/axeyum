# P2.10 — Breadth backlog (the remaining theory columns)

**Size:** per-item M–XL · **Depends on:** [P1.4 e-graph](../track-1-engine/P1.4-egraph.md),
[P1.5 CDCL(T)](../track-1-engine/P1.5-cdcl-t-loop.md) · **Enumerated, low priority**

## Goal
Enumerate — honestly and in one place — the SMT theory columns Z3/cvc5 support
that axeyum does **not** have at all yet, so "feature coverage" in the
[north star](../00-north-star.md) is a countable list, not a vibe. None of these
is on the critical path; this file exists so they are *tracked*, scoped, and
sequenced behind the keystones rather than forgotten.

## The columns
| id | theory | who has it | size | note / why it waits |
|---|---|---|---|---|
| T2.10.1 | **Sequences** (`Seq`) | Z3, cvc5 | L | generalizes strings (P2.7); do after strings are unbounded, sharing the same length/concat machinery |
| T2.10.2 | **Sets, bags, relations** | cvc5 (best), Z3 (sets) | L | finite sets first (cardinality + membership on the e-graph); relations later |
| T2.10.3 | **Separation logic** | cvc5 | XL | spatial conjunction / points-to; valuable for heap verification (pairs with P4.6 CHC) but a large standalone theory |
| T2.10.4 | **Finite fields** (`FF`) | cvc5 | M–L | Gröbner-basis driven; pairs with the NRA Gröbner work (P2.5 T2.5.3) — shared ideal machinery |
| T2.10.5 | **Co-datatypes** (infinite/streams) | cvc5, Z3 | M | extends datatypes (P2.9) with a coinductive acyclicity/uniqueness check |
| T2.10.6 | **Recursive function definitions** (`define-fun-rec`) | Z3, cvc5 | M | well-founded unfolding + the quantifier engine (P2.6); needed for many real benchmarks |
| T2.10.7 | **Nonlinear integer completeness gap** (NIA beyond the shared multivariate core) | Z3 | L | rides P2.5; tracked here so the NIA-specific gap is not lost inside the NRA frontier |

## Contract (applies to every item)
Each new theory ships as a `TheorySolver` on the CDCL(T) loop (P1.5), every
`sat` replay-checks, every new `unsat` route gets an independent checker or a
[trust-ledger](../track-3-proof-lean/P3.0-trust-ledger.md) entry, and a capability
row lands in `crates/axeyum-solver/src/capabilities.rs`. No item starts before the
e-graph + CDCL(T) keystones are real and at least one lazy theory (P2.3 EUF)
validates the loop.

## Phase exit criteria
- This table stays current: as each theory lands it moves to its own phase doc +
  a capability row; as new Z3/cvc5 columns are discovered they are added here.
- "Feature coverage" in the north star can be evaluated against an explicit,
  countable list of remaining columns, not a guess.
