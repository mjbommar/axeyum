# P5.3 — Kernel-shaped obligations: memory regions, 2-safety, FSM refinement

Status: ✅ bounded v1 complete (ADR-0322). The residuals named per family remain
future, separately evidence-gated work.

## Goal

The three obligation families a seL4-inspired Rust kernel/network stack
actually needs, each chosen because it is **finite, BV/array-shaped, and
already inside axeyum's strong fragment**: (a) bounded memory regions and
page-table-style index math, (b) 2-safety hyperproperties by self-composition
(constant-time / secret-independence), (c) protocol-FSM refinement (spec
machine ≃ reflected implementation).

## Why / leverage

This is where Track 5 differentiates on *substance*, not just trust: the
ecosystem's own first verification target (Asterinas/vostd) is the page-table
module; constant-time verification of kernel/crypto primitives is a high-value
niche that is decidable and badly served; and the FSM refinement rung is the
network-stack story (the spec-side toolkit already ships —
`protocol_toolkit.rs`, PDR/k-induction). Self-composition is nearly free in our
architecture: the cross-IR suites already lower two functions over shared
symbols in one arena; 2-safety is the same trick with one function twice.

## Dependencies

- P5.1 (front end; T5.1.5 memory for (a)).
- Shipped: `Sort::Array`/QF_ABV, `TransitionSystem` + `prove_safety_pdr` /
  `prove_safety_k_induction` / `bounded_model_check`, the protocol-FSM toolkit.
- (b) and (c) do not depend on T5.1.5 and can start immediately after T5.1.1.

## Phase exit criteria

- Each family has ≥1 committed, green, replayed/certified example on reflected
  code (not hand-built terms), with wall-times recorded.
- The refuted variants exist for each family (a leaky compare, a broken walk, a
  skipping FSM) — the prover must be demonstrably discriminating here too.

Bounded v1 exit accepted by ADR-0322: all three families have a committed green
positive and negative example with recorded wall observations, and the catalog
makes the source-authenticity and claim-boundary differences explicit.
