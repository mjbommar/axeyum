# P5.2 — Contracts & modular verification

## Goal

`#[requires(..)]` / `#[ensures(..)]` attributes on Rust functions that compile
into hypotheses/goals over the reflected terms, and **modular verification**:
each function is verified against its contract *assuming its callees'
contracts*, making function calls a composition rule instead of an inlining
problem.

## Why / leverage

This is the architectural unlock, not a convenience. ADR-0296/0297 now cross
one exact scalar LLVM boundary, but the general reflector still cannot compose
the checksum module without body-derived functional summaries (its earlier MIR
result relied on the inliner). SPARK and Verus both won adoption with exactly
this contract shape. Contracts keep every obligation **finite and decidable** —
the callee's `ensures` is just another hypothesis — which is what separates
this from ghost-code deductive systems. It also gives `#[verify]` its graduation
path: today's "no-annotation panic-freedom" stays the zero-config default;
contracts are the opt-in for cross-function claims.

## Dependencies

- P5.1 (reflected functions to attach contracts to; T5.1.1's API).
- The existing `axeyum-verify` proc-macro surface
  (`crates/axeyum-verify-macros/`) — extend, don't fork.

## Phase exit criteria

- A multi-function example (the checksum module is the canonical one) verifies
  modularly on both IRs, with contract violations producing replayed witnesses.
- The modular-vs-inlined differential gate is committed and green.
- The annotation story is documented: zero-annotation panic-freedom remains the
  default; contracts are additive.

ADR-0315 deliberately establishes the runtime panic-composition rule before
annotation lowering. It does not let source syntax silently define unwind or
postcondition semantics.

ADR-0316 supplies the documented additive annotation UX for one source-local
straight-line scalar body. The phase still needs an authenticated bridge from
that source contract to the already checked MIR/LLVM modular summaries; source
syntax alone cannot establish compiler-body identity. ADR-0317 preregisters the
smallest such MIR bridge over one total annotated wrapping function, with an
owning-Cargo-build artifact and an exact hand-built-summary differential before
implementation.
