# P5.2 — Contracts & modular verification

## Goal

`#[requires(..)]` / `#[ensures(..)]` attributes on Rust functions that compile
into hypotheses/goals over the reflected terms, and **modular verification**:
each function is verified against its contract *assuming its callees'
contracts*, making function calls a composition rule instead of an inlining
problem.

## Why / leverage

This is the architectural unlock, not a convenience. The reflector cannot cross
function boundaries today (the checksum module worked because the MIR inliner
had already composed the functions); SPARK and Verus both won adoption with
exactly this contract shape. Contracts keep every obligation **finite and
decidable** — the callee's `ensures` is just another hypothesis — which is what
separates this from ghost-code deductive systems. It also gives `#[verify]` its
graduation path: today's "no-annotation panic-freedom" stays the zero-config
default; contracts are the opt-in for cross-function claims.

## Dependencies

- P5.1 (reflected functions to attach contracts to; T5.1.1's API).
- The existing `axeyum-verify` proc-macro surface
  (`crates/axeyum-verify-macros/`) — extend, don't fork.

## Tasks

| ID | Task | Key references | Size | Exit |
|---|---|---|---|---|
| T5.2.1 | **Contract surface + lowering**: `#[requires]`/`#[ensures]` (expressions over params/return in the supported scalar/array fragment) parsed by `axeyum-verify-macros` and lowered to terms over the reflected function's symbols | `crates/axeyum-verify-macros/src/`, `crates/axeyum-verify/src/verify.rs` | M | a contracted function verifies (ensures proved under requires) and a violated contract yields a replayed witness |
| T5.2.2 | **Modular calls**: at a `Call` terminator (MIR) / `call` with result (LLVM), assert the callee's `requires` at the call site (an obligation), assume its `ensures` for the result (havoc + hypothesis); recursion declined honestly in v1 | P5.1 executors | M–L | the checksum module re-proves *without* relying on the inliner: `cksum_pair` verified against `sum16`'s contract |
| T5.2.3 | **Panic contracts**: `ensures(no_panic)` as the default obligation; callee panic conditions propagate to the caller's panic term (so cross-function panic-freedom composes) | `reflect_mir_into_checked` panic terms | S–M | a two-function chain's panic-freedom proves modularly and matches the inlined proof |
| T5.2.4 | **Contract soundness gate**: differential test — modular verdict vs inlined-reflection verdict on a corpus where both are computable; any divergence is a bug in the composition rule | `tests/cross_ir_equivalence.rs` patterns | M | committed gate, DISAGREE=0 between modular and inlined routes |

## Phase exit criteria

- A multi-function example (the checksum module is the canonical one) verifies
  modularly on both IRs, with contract violations producing replayed witnesses.
- The modular-vs-inlined differential gate is committed and green.
- The annotation story is documented: zero-annotation panic-freedom remains the
  default; contracts are additive.
