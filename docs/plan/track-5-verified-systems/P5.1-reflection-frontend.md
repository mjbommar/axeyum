# P5.1 â€” The reflection front end (crate-ify MIR + LLVM reflection)

## Goal

Turn the prototype reflectors in `crates/axeyum-verify/tests/reflect_common/`
into a real, reusable front end: parse the **actual artifacts of a `cargo`
build** (rustc MIR text, LLVM `.ll`) with full-fidelity parsers, reflect a
documented Rust/LLVM fragment into `axeyum-ir` terms plus **panic-condition
terms**, and expose the entry points the rest of the track builds on.

## Why / leverage

Everything in Track 5 flows through this: contracts (P5.2) attach to reflected
functions, kernel obligations (P5.3) are goals over reflected terms, the fuzz
loop (P5.4) uses reflections as oracles, and the external target (P5.5) is
"point the front end at their code." The prototypes proved the *semantics*
work â€” 34 green test binaries, millisecond proofs; what's missing is
robustness (real compiler output, not hand-curated fixtures), coverage
(calls, loops, memory), and a consumable API.

## Phase exit criteria

- A named public API reflects the supported fragment of **both** IRs from real
  build artifacts, returning `(value, panic)` term pairs.
- The prototype capability set (CFG, checked arithmetic, bounds, division,
  casts, multi-param, byte arrays) is preserved under the migration with all
  suites green, plus loops and bounded memory landed.
- The supported fragment, the UB stance, and the unsupported-construct behavior
  (loud decline, never silent misreflection) are documented in one place.
