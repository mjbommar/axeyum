# P5.4 — The fuzzing loop: reflections as oracles, witnesses as seeds

## Goal

Productize the verification↔fuzzing hybrid the prototypes exercised by hand:
**reflected terms as differential-fuzzing oracles** (real execution vs
reflection, DISAGREE=0), **solver countermodels as seed corpora** aimed at
exact boundary conditions, and an honest **`unknown` → directed-fuzz handoff**
for whatever the prover declines.

## Why / leverage

The goal statement is "formal verification *and fuzzing* in a natural way" —
and the two are one loop here, not two tools. Every prototype suite already
closes it manually: solver finds the witness → `catch_unwind` replays it →
neighbors confirmed clean. No current Rust tool offers "proofs where decidable,
solver-directed fuzz where not, one harness"; this phase is mostly *packaging*
of demonstrated capability, which is why it is M-sized and parallelizable with
P5.2.

## Dependencies

- P5.1 (the reflection API; T5.1.1 suffices to start).
- Shipped: witness replay patterns (`checked_*` suites), the deterministic
  differential fuzz (`cross_ir_equivalence.rs`), `axeyum-verify`'s
  witness-to-`#[test]` rendering (`reproduce`).

## Tasks

| ID | Task | Key references | Size | Exit |
|---|---|---|---|---|
| T5.4.1 | **Oracle harness**: given a reflected function + the real `fn`, run deterministic differential fuzz (seeded LCG, corner values, width-aware masks) with DISAGREE=0 reporting — one call, replacing the per-suite hand-rolled loops | `tests/cross_ir_equivalence.rs::differential_fuzz_*`, `tests/checksum_module.rs` | S–M | the five hand-rolled fuzz loops in the prototype suites collapse onto the harness; committed gate |
| T5.4.2 | **Witness corpus**: countermodels (panic witnesses, equivalence refutations, contract violations) exported as a deterministic seed corpus + generated `#[test]`s via the existing `reproduce` renderer | `axeyum-verify` witness rendering; `checked_division.rs`/`checked_bounds.rs` replay blocks | M | a witness found by the solver lands as a committed regression test automatically |
| T5.4.3 | **`unknown` handoff**: when a goal times out / leaves the fragment, emit the goal's input-space description (bounds, guards already proved) as a fuzz target spec instead of a bare `unknown` | `SolverConfig` timeouts; route-trace telemetry (ADR-0050) | M | an artificially hard goal produces a runnable directed-fuzz harness + an honest report distinguishing *proved* / *refuted+replayed* / *fuzzed-only* |
| T5.4.4 | **Coverage accounting**: the report separates proof-covered input space from fuzz-sampled space (never conflated) — the honesty layer the scoreboard discipline requires | bench-results conventions | S | report format documented + one committed example |

## Phase exit criteria

- One harness call gives: proofs where decided, replayed witnesses where
  refuted, directed fuzz where unknown — with the three outcomes reported
  distinctly and DISAGREE=0 between reflection and execution everywhere
  sampled.
